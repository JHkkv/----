# novel-writing 技能设计文档

## 概述

一个覆盖小说写作全流程的 AI 辅助技能包，包含续写、润色、设定管理、设定共创、自动汇总六大能力。采用单一技能 + 引用文件架构，写作前强制输出设定核查报告以解决"续写偏离前文设定"的核心痛点。

## 文件结构

```
skills/novel-writing/
  SKILL.md              # 核心工作流（所有能力模块的流程定义）
  style-guide.md        # 文风特征库（从样章自动提取 + 用户手动规则）
  settings-schema.md    # 设定模板与一致性检查清单
  chapter-template.md   # 章节元数据模板
```

## 能力模块

### 1. 续写（核心）

7 步工作流，分步确认：

| 步骤 | 名称 | 说明 | 确认点 |
|------|------|------|--------|
| 1 | 上下文收集 | 自动读取最近章节、设定文件、当前伏笔、人物状态 | 无（自动） |
| 2 | 设定核查报告 | **强制步骤**，列出当前境界/位置/伏笔/时间线约束 | 用户确认理解 |
| 3 | 剧情讨论 | 提出2-3个走向选项，或用户指定方向 | 用户选择方向 |
| 4 | 大纲拟定 | 章节结构（分节）、每节关键事件、字数预估 | 用户确认大纲 |
| 5 | 正文写作 | 参照文风特征库写出正文，标注新增设定点 | — |
| 6 | 审阅 | 设定一致性复查、文风匹配自查、用户反馈修改 | 用户确认定稿 |
| 6.5 | 字数调整 | 不足则扩展，超出则精简，达到目标字数 | 用户确认 |
| 7 | 归档汇总 | 写入章节文件 + 追加全本 + 更新设定 + 更新索引 | 无（自动） |

### 2. 设定管理

双轨机制：
- **设定文件**（权威来源）：`材料_设定.txt`，包含世界观、人物谱、伏笔清单、时间线
- **memory 索引**（快速检索）：最新章节号、人物状态、关键伏笔状态

写作前自动执行一致性检查：

| 检查项 | 说明 |
|--------|------|
| 境界一致 | 人物当前境界是否与前文匹配 |
| 位置一致 | 人物当前所在地是否合理 |
| 因果一致 | 新事件是否与前文因果矛盾 |
| 伏笔状态 | 是否有已埋未收的伏笔需要在本次涉及 |
| 时间线 | 事件发生顺序是否合理 |

### 3. 润色

工作流：
1. 用户指定润色段落
2. 对照文风特征卡诊断偏离（句式、对话比例、描写密度、节奏）
3. 输出诊断报告 + 修改建议
4. 用户确认修改方向后逐项修改
5. 用户审阅 → 归档

**审阅冲突检测**：当用户修改意见与前文设定矛盾时，主动提醒并说明冲突点。

**自学习能力**：累计采纳超过 20 次后，同类修改可跳过用户确认直接执行。

### 4. 设定共创

工作流：
1. **引导提问**：用户提出想法后，一次一个关键问题（不超过3个），帮助理清思路
2. **方案生成**：基于已有设定给出 2-3 个具体方案，附推荐理由和与主线的关联说明
3. **落地整合**：用户选择后，更新设定文件，标注与现有伏笔的关联，检查新矛盾

### 5. 自动汇总

章节审阅通过后自动执行：
1. 写入独立章节文件（含元数据头部）
2. 追加到全本汇总文件
3. 更新设定文件（新人物/新伏笔/境界变化）
4. 更新 memory 索引

### 6. 贯穿能力

- **设定一致性主动检查**：所有涉及内容修改的操作前执行
- **审阅冲突提醒**：用户意见与前文矛盾时主动提醒
- **自学习跳过确认**：连续采纳超阈值后同类修改自主执行

## 互动模式

混合模式：执行指令 + 主动发现问题并提问

- 执行用户明确指令（续写、润色、查设定等）
- 主动发现设定矛盾、文风偏离、节奏问题并提醒
- 以编辑/搭档角色协作，而非单纯工具

## 章节文件元数据格式

```
---
章节: 第七十二章
标题: 林家险境
字数: 4200
视角人物: 柳绝
新增人物: 时遗族男子
境界变化: 无
新增伏笔: 时遗族因果线
关键事件: 万道苑被袭, 柳天被黑焰所伤
状态: 定稿
审阅次数: 3
---
```

## 核心原则

- 动笔前必须输出设定核查报告
- 分步确认，不一次性输出长文
- 主动发现问题并提醒，不只被动执行
- 连续采纳超过 20 次可自主跳过确认
- 设定文件为权威来源，memory 为快速索引
