## 一、产品概述

**树洞漂流瓶**是一个以"情绪倾诉"为核心的在线匿名社区。用户将心事写入瓶中投入大海，也可以在海面上打捞他人的漂流瓶。核心隐喻：**把你的不开心送走**。

### 核心差异化
- 渐进式 MBTI 性格预测 → 匹配心境相似的人
- 深海夜光美学 → 温暖治愈的视觉体验
- 仅瓶主可见的评论 → 安全的情绪空间

## 四、设计变体方案对比与选择

以下对关键设计维度进行了 20+ 方案对比，择优标注 ✅。

### 4.1 视觉风格（6 方案）

| # | 方案 | 配色基调 | 情绪传达 | 选择 |
|---|------|---------|---------|------|
| 1 | 深海夜光 | 深蓝#0a1628 + 暖金#e8b86d | 孤独中温暖 | ✅ |
| 2 | 温柔花海 | 粉紫#fdf2f8 + 深紫#7c3a6e | 少女感拥抱 | |
| 3 | 禅意纸笺 | 米白#f5f0e8 + 棕#4a3728 | 极简安静 | |
| 4 | 星空幻想 | 深紫#0c0e1a + 紫蓝#a088e0 | 奇幻治愈 | |
| 5 | 极光冰洋 | 墨绿#0a1a1a + 极光绿#40e0d0 | 清冷孤独 | |
| 6 | 暮色沙滩 | 暗橙#1a1008 + 金粉#f4a460 | 落日余晖 | |

**选择理由**: 深海夜光在孤独感与温暖感之间取得了最佳平衡，暗蓝海面是天然的大面积留白，暖黄瓶光形成强对比焦点。

### 4.2 瓶子造型（5 方案）

| # | 方案 | 特征 | 选择 |
|---|------|------|------|
| 1 | 红酒瓶型 | 直筒身、长瓶颈、瓶肩锐利 | |
| 2 | 圆腹瓶型 | 腹部膨大、短瓶颈、软木塞 | ✅ |
| 3 | 六角药瓶 | 几何棱角、金属盖 | |
| 4 | 试管型 | 细长直筒、橡胶塞 | |
| 5 | 陶罐型 | 不透明、粗陶纹理 | |

**选择理由**: 圆腹瓶型最接近经典"瓶中信"意象（参考用户提供的实物图），透明玻璃 + 可见纸卷是核心互动线索——用户看到纸卷就知道里面有内容可读。

### 4.3 瓶子动画（5 方案）

| # | 方案 | 动画描述 | 选择 |
|---|------|---------|------|
| 1 | 原地浮动 | 仅上下 bobbing | |
| 2 | 水平漂移 | 左右平移 + 微旋转 | |
| 3 | 椭圆轨道 | 画椭圆圈 | |
| 4 | 水流路径 | 水平漂移 + 随机波浪 + bobbing | ✅ |
| 5 | 物理模拟 | 基于流体力学 | 过度工程 |

**选择理由**: 水流路径方案在视觉真实感与实现复杂度之间最佳平衡。三瓶以不同速度/幅度/相位沿水平方向漂移（模拟海流），叠加独立垂直 bobbing（模拟波浪），再加 -28°~-35° 倾斜漂浮姿态。Framer Motion 的 `useMotionValue` + `useTransform` + `useSpring` 天然支持。

### 4.4 月亮设计（4 方案）

| # | 方案 | 实现 | 选择 |
|---|------|------|------|
| 1 | 多 div 径向渐变光晕 | 4 个 div 叠加 | 有方框痕迹 ❌ |
| 2 | 单 div + box-shadow | 原生 CSS | ✅ |
| 3 | SVG 渐变月亮 | 精确但重 | |
| 4 | PNG/CSS sprite | 静态图 | |

**选择理由**: box-shadow 方案零额外 DOM、天然圆润无边界的柔光扩散，且动画性能最佳（只触发 composite）。

### 4.5 页面布局（4 方案）

| # | 方案 | 结构 | 选择 |
|---|------|------|------|
| 1 | 单一海面 | 一页到底，瓶子混在海面上 | |
| 2 | 双模式切换 | 顶栏切换「看」/「写」模式 | ✅ |
| 3 | 三栏布局 | 左书写 + 中海面 + 右记录 | |
| 4 | Tab 式 | 底部导航切换 | |

**选择理由**: 双模式保持海面作为唯一视觉焦点，切换简单明了，避免信息过载。


### 5.2 主页交互流

```
┌──────────────────────────────────────────────────┐
│  顶栏: Logo | MBTI闪烁提醒 | [看别人|写自己] | 昵称  │
├──────────────────────────────────────────────────┤
│                                                    │
│              ✦   🌙 明月  ✦                        │
│                                                    │
│       🍾(远)    🍾(中)      🍾(近·主瓶)            │
│     ═══════════ 水面  ═══════════                  │
│                                                    │
│           🌊 把你的不开心送走                        │
│                                                    │
└──────────────────────────────────────────────────┘
```

**「看」模式**:
- 3 个瓶子以不同路径顺水流漂移
- 光标悬浮 → 瓶子放大 1.15x + 光晕增强 + 漂移减速
- 点击 → 瓶子平移至屏幕中央 → 放大 → 模态展开内容
- 模态内: 阅读内容 + 评论输入 + 「仅瓶主可见」标识

**「写」模式**:
- 海面淡出 → 书写区淡入（保留背景）
- 文本输入 + 可见范围切换 + 「投出漂流瓶」按钮
- 投出 → 预览弹窗（双重确认） → 确认 → 瓶子弹跳入海动画

### 5.3 可见范围逻辑

| 可见性 | 谁能在海上看到 | 谁能评论 | 评论谁可见 |
|--------|--------------|---------|-----------|
| 公开 + 允许评论 | 所有人 | 所有人 | 仅瓶主 |
| 公开 + 禁止评论 | 所有人 | 无人 | N/A |
| 仅自己可见 | 无人 | 无人 | N/A |

## 六、MBTI 系统设计

### 6.1 三阶段策略

```
阶段一 (用户 ≤ 20): AI 情绪分析 + 选择题 → 双重信号
阶段二 (用户 > 20): 纯选择题计分 → 规则引擎
任意时刻:        用户直接填写已知 MBTI → 置信度 = 1.0
```

### 6.2 渐进式答题逻辑

- 题库: 4 维度 × 15 题 = 共 60 题
- 每轮: 随机抽取 2-3 题，优先覆盖低置信度维度
- 计分: 每维度独立计分（-1.0 ~ +1.0 连续值）
- 映射: 分数符号决定维度倾向（E vs I, S vs N, T vs F, J vs P）
- 置信度: 基于已答题数 / 建议总题数 + 分数绝对值一致性

### 6.3 AI 情绪分析（阶段一，≤ 20 人）

- 输入: 用户书写的瓶子内容文本
- 调用: Claude API
- Prompt: "根据以下情绪倾诉文字，推测 MBTI 四个维度的倾向，给出连续值分数"
- 输出: 四维度分数 + 置信度
- 合并: AI 分数 × 0.3 + 选择题分数 × 0.7（如果两者都有）

### 6.4 MBTI 首次体验

```
进站 → 弹窗「欢迎！完成MBTI可以更懂你」
     → [做几道题] → /mbti → 答 2-3 题 → 完成页 → 回首页
     → [已知MBTI] → /mbti/direct → 选 16 型 → 回首页
     → [先跳过]   → 直接进首页（无匹配能力）
```

## 九、非功能需求

### 性能
- LCP < 2.5s (灯塔图片优化 + 字体子集)
- Framer Motion 动画保持 60fps (仅使用 transform/opacity)
- API 响应 < 200ms (p95)

### 安全
- cookie_token 使用 crypto.randomUUID()
- 瓶子内容 max 2000 字符，评论 max 500
- SQL 参数化查询 (Prisma 默认)
- XSS: React 默认转义 + 瓶子内容不渲染 HTML
- 游客 token 30 天过期

### 可访问性
- 所有按钮有 aria-label
- 颜色对比度 ≥ 4.5:1
- 键盘可导航（Tab + Enter）

### 部署
- Vercel 自动部署（Git push → deploy）
- 环境变量: DATABASE_URL, CLAUDE_API_KEY (阶段一)
- 用户满 20 人后移除 CLAUDE_API_KEY 依赖
