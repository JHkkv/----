## 缓存优化工作准则（Cache-First Protocol）

> **目标**：最大化 Anthropic prompt cache 命中率，减少重复 token 消耗，同时保持创作质量零下降。

### 原理

Anthropic prompt cache 缓存对话开头的共同前缀。每次会话加载**相同的静态文件、相同的顺序**，缓存命中率最高。静态设定始终在对话开头被缓存，只有尾部的动态内容（新章节）每次不同。

### 上下文加载顺序（每次会话必须遵守）

| 优先级 | 内容 | 文件 | 策略 |
|--------|------|------|------|
| 1 | 设定总纲（缓存锚点） | `写作/历史小说/小说设定总纲.md` | **每次必读，全文加载**——这是缓存的核心锚点，内容不变则整个前缀被命中 |
| 2 | 章节索引 | `结构/章节索引.md` | **全文加载**——替代读取全量章节文件，快速定位 |
| 3 | 写作风格 | `设定/写作风格.md` | **全文加载**——风格约束每会话需要 |
| 4 | 相关人物卡片 | `人物/*.md` | **按需加载**——只读本次章节涉及的人物（≤3人） |
| 5 | 前一章全文 | `章节_第21章起/第XXX章_标题.md` | **仅读最近 1 章**——用于承接文气和伏笔 |
| 6 | 项目日志 | `项目日志/最新一份` | **全文加载**——会话交接信息 |

### 硬性规则

1. **禁止**一次读取超过 2 章的完整文本——用章节索引的摘要替代
2. **禁止**重复读取本会话已读过的文件——已读内容在对话中可直接引用
3. 新章节写作前，只加载「前一章全文 + 当前章大纲」
4. 设定查询优先用章节索引定位，不盲读全量文件
5. 秦史参考材料（`秦史参考/`）仅在涉及历史考据时按需读取单份，**永不批量加载**
6. 每次文件读取前自问：「这个信息在章节索引摘要中能否找到？」——能则不读全文

### 质量保障红线

以下规则**不可被缓存优化覆盖**：
- 写新章节时，**必须**读取前一章全文（文气衔接）
- 涉及伏笔推进时，**必须**读取相关伏笔的原始出处章节
- 人物对话涉及特定角色时，**必须**读取该角色的人物卡片
- 设定变更时，**必须**先读取设定总纲确认当前状态

## 财务Agent 协议（Token Budget Agent）

> **角色**：会话的 token 资源管理者。在不影响创作质量的前提下，最大化 token 利用效率。

### 会话启动检查清单

每次写作会话开始时，执行以下步骤：

1. **确认缓存锚点** — 设定总纲是否为本次第一个读取的文件？（是→缓存大概率命中）
2. **列出加载清单** — 在开始创作前，先列出本次需要读取的文件清单（不超过 6 个文件）
3. **估算 token 预算** — 将 token 分配为：

| 用途 | 占比 | 说明 |
|------|------|------|
| 设定+索引加载 | ~25% | 缓存锚点，后续会话边际成本趋近于零 |
| 创作输出 | ~55% | 新章节文本，核心产出 |
| 修改+同步 | ~15% | 更新索引、同步设定、写项目日志 |
| 缓冲 | ~5% | 应对意外的长章节或复杂设定查询 |

### 会话结束报告

每次写作会话结束时，输出简要报告：

```
📊 会话财务报告
- 本次加载文件：[文件列表]
- 创作产出：第XXX章（约XXXX字）
- 缓存优化判断：[命中/未命中，原因]
- 下次会话建议：[可跳过的文件 / 需要预加载的内容]
```

### 高频场景速查

| 场景 | 加载什么 | 不加载什么 |
|------|----------|------------|
| 写新章节 | 设定总纲 + 章节索引 + 前一章 + 相关人物卡片 | 秦史参考、更早的章节全文 |
| 修改旧章节 | 设定总纲 + 该章全文 + 前后章索引摘要 | 人物卡片（除非涉及新设定） |
| 设定变更 | 设定总纲 + 相关人物/系统卡片 + 章节索引 | 章节全文 |
| 伏笔检查 | 章节索引 + 伏笔出处章节摘要 | 设定总纲（可跳过） |
| 新会话交接 | 设定总纲 + 章节索引 + 最新项目日志 | 旧项目日志 |

## 工作流

1. **查看小说** → 从 `镇秦.md` 进入，按人物/系统/结构/设定导航
2. **处理新章节** → 在 Obsidian 中打开 `结构/章节索引`，查看最新进度
3. **修改设定** → 在对应页面直接编辑，Claude 可辅助维护双向链接

### 新章节同步（写入后必须执行）

当用户创作新章节后，**自动完成以下同步**：

1. **更新章节索引** — `结构/章节索引.md` 新增章目（标题 + 核心内容摘要）
2. **更新状态信息** — 总索引 `镇秦.md` 的章数状态
3. **检查设定一致性** — 如有新增人物/系统/设定元素，同步更新对应页面
4. **检查大纲** — 如实际章节走向与 `小说大纲_前100章.md` 不一致，提示用户确认
5. **归档源文件** — 如源文件位于写作目录外，同步到 `写作/历史小说/章节_第21章起/`

### 设定修改同步

当用户修改设定（人物、系统、世界观）时：

1. 先更新源文件 `写作/历史小说/小说设定总纲.md`（唯一权威来源）
2. 再同步到 `镇秦小说/` 对应页面
3. 检查是否与已写章节冲突，有冲突主动提示
