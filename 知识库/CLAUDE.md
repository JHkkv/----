# 知识库 — CLAUDE.md

## 目录结构

```
知识库/
├── raw/          # 待处理的原始材料
├── wiki/         # 已内化的概念页（双向链接）
├── output/       # AI 生成的输出内容
├── cloud.md      # AI 处理协议（核心规则文件）
├── .processing-log.md  # 处理日志，标记哪些文件已处理
└── wiki/index.md # Wiki 索引，概念地图
```

## 小说同步

处理小说相关内容时，必须遵循 [novel-sync.md](novel-sync.md) 中的协议：
- 新章节写入后 → 自动更新章节索引、总索引状态、检查大纲一致性、检测人物/设定变更
- 设定修改后 → 先更新源文件总纲，再同步 Obsidian 页面
- 小说 Obsidian 仓库：`f:/测试工具/镇秦小说/`
- 源文件目录：`f:/测试工具/写作/历史小说/`

## 核心规则

**处理知识库时，必须遵循 [cloud.md](cloud.md) 中的协议：**

1. **增量处理** — 仅处理 `raw/` 中未在 `.processing-log.md` 标记的新文件
2. **概念提取** — 每份材料提取 3-8 个核心概念，存入 `wiki/`
3. **双向链接** — 使用 `[[wiki/概念名]]` 格式，新增概念必须回链已有概念
4. **溯源引用** — 每个知识点标注来源 `> 来源：raw/文件名`
5. **闭环回存** — 用户说"回存"时，将 AI 回答存入 `raw/`

## 工作流

当用户说"处理知识"或"处理 [文件名]"时：
1. 读取 cloud.md
2. 读取 .processing-log.md 确认待处理文件
3. 逐个处理新文件，严格按协议执行
4. 完成后报告：处理 X 个文件，提取 Y 个新概念，关联 Z 个已有概念
