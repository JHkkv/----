# 多Agent并行头脑风暴

## 定义

一种结构化AI协作工作流：将10位专业Agent并行启动，各自从第一性原理出发进行独立分析，然后交叉验证、融入本地实战案例，最终产出可执行的汇报和执行路线图。

## 核心流程

```
议题定义 → 10Agent并行分析 → 交叉验证 → 本地案例量化评分 → PPT输出 → 双主管对比 → 增强版迭代
```

## 两个版本

### v1.0 (初版)
- Hermes Agent 主导，希腊字母命名 (α~κ)
- 三回合制：独立分析 → 交叉辩论 → 收敛投票
- 输出：27页PPT + 方法论框架
- 特点：展示了Agent间的辩论过程

### v2.0 (增强版) — 2026-07-05
- 融入3个本地项目源码实地探查
- 增加量化ROI预估和100天三层递进路线图
- 补强CC方的TCO/风险矩阵弱点
- 输出：24页增强版PPT + 完整Obsidian知识库同步
- 新增内容：树洞漂流瓶项目分析、AI效益公式、三项目财务对比

## 关键差异化

1. **必须融入用户本地项目**作为实战验证案例
2. **AI适合度量化评分**：频率×复杂度×容错度×规模化（每项1-5分，总分≥14才值得上AI）
3. **100天可执行路线图**：D0-7→D8-21→D22-42→D43-70→D71-100，每阶段有Go/No-Go标准
4. **具体数字**：金额（万元）、百分比、周期，杜绝模糊描述
5. **双主管对比**：存在另一位主管汇报时，必须做逐维度对比分析
6. **Obsidian双向链接归档**：wiki知识沉淀 + raw原始记录 + output输出报告

- 12个月ROI预估表

## 关键共识（20位AI专家交叉验证）

1. **场景选择 > 技术能力**（10/10全票）
2. **小切口·深验证·快迭代**（9/10）
3. **人机协作 > 全自动替代**（8/10）
4. **付费POC > 免费验证**（双方独立得出）
5. **数据准备占60-80%工作量**
6. **组织摩擦是最大非技术失败原因**
