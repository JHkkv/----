# 树洞漂流瓶项目分析

## 基本信息

| 维度 | 详情 |
|------|------|
| **路径** | `f:/测试工具/项目/treehole-bottle` |
| **技术栈** | Next.js 14 · TypeScript · Prisma + PostgreSQL · TailwindCSS · framer-motion |
| **AI集成** | Claude Haiku API (情绪分析 → MBTI人格推断) |
| **AI就绪度** | 60% — 已集成Claude但有提升空间 |
| **当前状态** | 开发中，L1部门级部署 |

## 数据模型

4张核心表：
- `users` — 用户（昵称/游客/邮箱/MBTI类型/评分/置信度）
- `bottles` — 漂流瓶（内容/可见性/样式/打开次数）
- `comments` — 评论
- `mbti_answers` — MBTI答题记录（8维评分）

## AI能力

Claude Haiku 分析用户情绪文本 → 推断 MBTI 8维评分(0-1) → 16型人格分类 → 人格匹配社交

### 关键设计亮点
- **隐形AI**：用户在使用情感社区，不是"用AI"——完美契合UX最佳实践
- **渐进式MBTI**：从文本情绪推断 → 问卷验证 → 人格匹配，逐层深入
- **低成本模型**：Claude Haiku (最便宜)，情感分析不需要强模型

## AI适合度评估

频率 4 × 复杂度 4 × 容错 3 × 规模 4 = **15分** ✅ (值得投入)

## 优势

1. **情感数据 = 终极护城河**：独占的用户情感文本+MBTI标注，不可复制
2. **网络效应潜力**：人格匹配 → 社交推荐 → 更多用户 → 更多数据
3. **低边际成本**：Claude Haiku 极低成本，边际用户几乎零增量
4. **已有用户基础**：10万+情绪文本积累

## 风险

1. **隐私合规**：情感数据涉及敏感个人信息，需GDPR/个保法框架
2. **AI准确率争议**：从短文本推断人格，学术争议（需标注置信度）
3. **模型供应商锁定**：依赖Claude API，无自建替代方案

## 推荐策略

| 优先级 | 行动 | 投入 | 周期 |
|--------|------|------|------|
| P0 | 建立数据合规框架（用户授权/分类分级/脱敏） | 0.5人月 | 2周 |
| P1 | AI人格匹配v2.0（置信度可视化+用户纠偏） | 1人月 | 4-6周 |
| P2 | 人格匹配社交功能（相似/互补推荐） | 1-2人月 | 6-8周 |
