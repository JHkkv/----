# 小说章节同步协议

## 触发条件

当用户在小说创作过程中说以下任何内容时，自动执行本协议：

- "写第X章"
- "续写"
- "继续写"
- "新章节"
- 或在 `写作/历史小说/章节_第21章起/` 目录下发现新创建的文件

## 同步流程

### 一、新章节写入后

每一次章节创作完成后，**必须逐项执行**以下同步：

#### 1. 更新章节索引
- 文件：`f:/测试工具/镇秦小说/结构/章节索引.md`
- 操作：新增章目行（章号 + 标题 + 一句话核心内容）
- 如超出已有大纲范围（如第120章+），在对应卷/弧段下追加

#### 2. 更新总索引状态
- 文件：`f:/测试工具/镇秦小说/镇秦.md`
- 操作：更新"状态"行的章数（如"至第120章"）

#### 3. 检查大纲一致性
- 文件：`f:/测试工具/写作/历史小说/小说大纲_前100章.md`
- 操作：
  - 如章节走向与大纲一致 → 标记"按大纲：第X弧"
  - 如偏离大纲 → **主动提示用户**："这章走向与原大纲不同，是否更新大纲？"
  - 如超出大纲范围（101章+）→ 提示用户是否有新卷大纲需要补充

#### 4. 人物/设定变更检测
逐项检查新章中是否出现：
- 新人物 → 在 `镇秦小说/人物/` 建新页
- 人物关系变化 → 更新双方页面
- 新设定/新系统元素 → 在 `镇秦小说/系统/` 或 `镇秦小说/设定/` 更新
- 修为变化 → 同步更新 `人物/江述.md` 的修为历程
- 与现有设定冲突 → **主动提示用户**，不自动覆盖

#### 5. 源文件归档
- 如新章文件不在 `写作/历史小说/章节_第21章起/` 下 → 复制到该目录
- 命名规范：`第XX章_标题.md` 或 `.txt`

### 二、设定修改后

当用户修改设定（人物属性、系统参数、世界观规则）时：

1. **先更新源文件** `写作/历史小说/小说设定总纲.md`（唯一权威来源）
2. **再同步** `镇秦小说/` 对应页面
3. **回扫已写章节**：如新设定与已写内容冲突 → 列出冲突章节，提示是否修订

## 同步记录

每次同步后，在 `f:/测试工具/镇秦小说/.sync-log.md` 追加一行：

```
| 时间 | 操作类型 | 涉及文件 | 状态 |
```
