# Coze 一键网页解析工作流 — 手把手教程

## 🎯 目标
搭一个 Bot：输入一个网址，自动返回该网页的核心内容摘要。

---

## 📝 第一步：创建 Bot

1. 打开 [Coze.cn](https://www.coze.cn) → 点左侧 **「+ 创建Bot」**
2. 填写：
   - Bot名称：`网页速读助手`
   - Bot描述：`输入任意网址，自动提取并总结网页核心内容`
   - 图标：随便选一个
3. 点 **「确认」**

---

## ⚙️ 第二步：搭工作流

进入 Bot 编辑页 → 点顶部的 **「工作流」** 标签 → 点 **「+ 创建工作流」**

### 2.1 开始节点 — 接收网页链接

点击 **「开始」** 节点 → 添加输入变量：

| 变量名 | 类型 | 描述 |
|--------|------|------|
| `url` | string | 要解析的网页地址 |

### 2.2 插件节点 — 读取网页内容

1. 从左侧拖一个 **「插件」** 节点到画布
2. 搜索选中 **「Link Reader」** 插件（Coze 内置的免费网页读取插件）
3. 点击 Link Reader 节点 → 把 `url` 字段绑定到开始节点的 `url` 变量

> 💡 **没有 Link Reader？** 换成 **Code 节点** + 下面的备选方案（见文末）

### 2.3 LLM 节点 — 总结网页内容

1. 从左侧拖一个 **「大模型」** 节点到画布
2. 连接到 Link Reader 节点的输出
3. 设置如下：

| 配置项 | 值 |
|--------|-----|
| 模型 | 豆包 Lite（够用还免费） |
| 温度 | 0.3 |
| 输入 | 把 Link Reader 的 `content` 输出绑过来 |

**提示词（粘贴到"用户提示词"框）：**
```
你是一个专业的信息提取助手。请根据下面抓取到的网页内容，按以下格式输出总结：

📌 **标题**：（提取或概括网页的核心标题）

📝 **核心要点**（3-5 条）：
- 要点1
- 要点2
- ...

💡 **一句话总结**：（用一句话概括整篇文章说了什么）

🏷️ **关键词**：（5个以内的关键词/标签）

================================
网页内容：
{{上一步的content变量}}
```

### 2.4 结束节点 — 输出结果

1. 拖一个 **「结束」** 节点
2. 连接到 LLM 节点的输出
3. 添加输出变量：
   - 变量名：`summary`，类型：string，值：绑定 LLM 节点的输出

### 2.5 连接 + 试运行

```
开始(url) → 插件(Link Reader) → 大模型(总结) → 结束(summary)
```

点右上角 **「试运行」** → 输入测试网址：`https://36kr.com/p/123456`
看看输出！

---

## 🧪 第三步：发布测试

1. 工作流编辑页点右上角 **「发布」**
2. 回到 Bot 编辑页 → 在 **「人设与回复逻辑」** 里写：

```
你是网页速读助手。当用户发送网址时，调用"网页解析"工作流提取内容并总结。
如果用户没发网址，回复"请把网页链接发给我，我来帮你速读！"
```

3. 在 **「技能」** 区域 → 添加刚发布的工作流
4. 点右上角 **「发布」** → 选 **「豆包」** 渠道

## 🔧 备选方案：没有 Link Reader 插件时用 Code 节点

如果 Coze 插件市场搜不到 Link Reader，用 Code 节点代替：

1. 拖一个 **「Code」** 节点
2. 输入参数：`url`（string，绑定开始节点的 url）
3. 输出参数：`content`（string）
4. 编程语言选 **Python**
5. 粘贴以下代码：

```python
import requests
import re

def main(args):
    url = args.get('url', '')
    if not url:
        return {'content': '请提供有效的网址'}

    try:
        headers = {
            'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36'
        }
        resp = requests.get(url, headers=headers, timeout=15)
        resp.encoding = 'utf-8'

        # 去掉 script 和 style 标签
        text = re.sub(r'<script[^>]*>.*?</script>', '', resp.text, flags=re.DOTALL)
        text = re.sub(r'<style[^>]*>.*?</style>', '', text, flags=re.DOTALL)
        # 去HTML标签
        text = re.sub(r'<[^>]+>', '', text)
        # 去多余空行
        text = re.sub(r'\n\s*\n', '\n', text)
        # 截取前 8000 字（防 token 超限）
        text = text.strip()[:8000]

        return {'content': text}

    except Exception as e:
        return {'content': f'网页抓取失败：{str(e)}'}
```

> ⚠️ 注意：Code 节点的 requests 库在 Coze 上可直接用，无需安装。

## 📌 避坑提示

1. **变量绑定**：节点之间连了线 ≠ 变量自动对应，你还得在每个节点里手动选"引用 → 上一节点输出"
2. **试运行**：每搭一个节点就跑一次试运行，不要全搭完再测
3. **超时**：有些网站反爬，Code 节点可能超时。遇到这种情况就换成 Link Reader 插件，它能绕过大部分反爬
4. **Token 限制**：网页太长会超出模型限制。Code 节点里已经截了 8000 字，够用

## 🎓 学完后你可以怎么扩展？

学会了这个基础版本，你可以继续加功能：

- **多语言翻译**：在 LLM 节点后面加一个翻译节点，把英文网页翻译成中文
- **定时监控**：用 Coze 的定时触发器，每天自动检查某个页面的更新
- **存数据库**：加上飞书多维表格节点，把解析结果存到表格里归档
- **多网页合并**：输入 3 个网址，把它们的观点合并成一份对比分析

这些就是第五课会讲的内容，先把手头这个跑通 ✌️
