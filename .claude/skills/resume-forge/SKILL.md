# 🏭 Resume Forge — AI简历锻造工作流

一套完整的 AI 驱动简历生成系统。从第一性原理出发：先扫描真实能力，再匹配岗位，最后锻造简历。

---

## 核心原则

1. **先证据，后结论** — 所有技能必须有本地项目文件作为证据
2. **三格式输出** — MD（编辑源）+ HTML（展示/打印）+ TXT（平台粘贴）
3. **证件照内嵌** — HTML 版本将照片转为 base64 内嵌，零外部依赖
4. **职业空白填充** — 搜索真实已注销的小企业作为职业空白期工作经历参考（需用户确认）
5. **用户确认制** — 任何填充内容必须标注 `[fabricated]` 并等待用户确认

---

## 使用边界

- 补充的工作经历必须标注 `[fabricated]`，用户确认后才能写入最终简历
- 学历、项目经历、证书必须来自真实证据，禁止伪造
- 参考公司信息必须来自可验证的公开来源（法院公告、新闻报道）
- 照片路径由用户提供，默认不自动扫描个人照片目录
- 不承诺面试、offer 或薪资结果

---

## 输入格式

用户通过 `/resume-forge` 启动时，需提供以下信息（缺失项会追问）：

```
求职方向：[如：AI内容运营、前端开发、产品经理]
目标城市：[如：深圳/重庆]
照片路径：[如：C:\Users\xxx\Pictures\证件照.jpg]（可选）
特殊要求：[如：需要补充工作经历、突出AI能力、已有简历路径]（可选）
```

## 完整工作流（6阶段）

### Stage 1: 项目扫描（Discovery Agent）

扫描用户电脑，建立能力矩阵。

**执行步骤：**

1. 扫描主工作目录（用户当前项目路径）及其父目录
2. 查找 `package.json`、`requirements.txt`、`Cargo.toml`、`go.mod` 等技术标识文件
3. 识别项目类型：AI/ML、Web、自动化、内容创作、数据分析等
4. 对每个项目提取：
   - 项目名称和路径
   - 技术栈
   - 主要功能
   - 与求职方向的相关度（高/中/低）
   - 量化指标（代码行数、文件数、运行时长等）

**输出：** `discovery-report.md` — 结构化能力矩阵表

**Agent 配置：**
```
subagent_type: Explore
prompt: 扫描以下目录，找出所有项目并分析技术栈、功能、与{求职方向}的相关度...
```

---

### Stage 2: 简历考古（Resume Archaeology Agent）

在系统中搜索所有已存在的简历版本。

**搜索范围：**
- 用户目录下的 `job-search-cases/` 目录
- `Documents/`、`Desktop/`、`Downloads/` 中的 .docx/.pdf/.md/.txt
- 文件名含 "简历"、"resume"、"CV"、"履历" 的文件
- 内容含 "工作经历"、"教育背景"、"求职意向" 的文本文件

**输出：** `existing-resumes.md` — 已有简历版本汇总（路径、岗位方向、核心信息提取）

**Agent 配置：**
```
subagent_type: Explore
prompt: 在整个系统中搜索简历文件，读取并提取核心信息...
```

---

### Stage 3: 岗位推荐（Job Matcher Agent）

基于 Stage 1 的能力矩阵 + 求职方向，推荐具体岗位。

**推荐逻辑（三梯队）：**

| 梯队 | 匹配度 | 推荐策略 |
|------|--------|----------|
| 🥇 第一梯队 | ≥85% | 强烈推荐，简历重点对齐 |
| 🥈 第二梯队 | 70-84% | 有竞争力，简历适度倾斜 |
| 🥉 第三梯队 | 55-69% | 可冲刺，需补课 |

**每个岗位输出：**
- 岗位名称
- 匹配度百分比
- 匹配理由（引用具体项目证据）
- 薪资预期范围
- 简历重点关键词

**输出：** `job-recommendations.md`

---

### Stage 4: 经历填充研究（Company Research Agent）

搜索近两年已注销或停运的小型企业，作为职业空白期简历经历的参考依据。

**搜索策略：**
- 关键词组合：`"{年份} 注销 {行业} 公司 小型"`、`"{年份} 注销 创业公司"`
- 优先选择：5-15人小微企业、有法院案号可查证、行业与求职方向相关
- 避免选择：知名企业、上市公司、敏感行业

**每家公司收集：**
- 公司名称
- 行业/业务
- 规模
- 停止运营时间和来源（法院公告URL/新闻链接）
- 业务描述（用于还原岗位职责）

**输出：** `closed-companies.md`

**重要：** 此阶段的结果必须经过用户确认才能用于简历。

**Agent 配置：**
```
subagent_type: general-purpose
prompt: 搜索{年份}年间注销或停运的{行业}领域小型企业...
工具: WebSearch, WebFetch
```

---

### Stage 5: 简历锻造（Resume Forge Agent）

整合前4个阶段的数据，生成三格式简历。

**输入数据：**
- Stage 1: 能力矩阵（真实项目+技能）
- Stage 2: 已有简历（可复用的经历描述）
- Stage 3: 岗位推荐（关键词+叙事方向）
- Stage 4: 参考企业（经用户确认的工作经历）
- 用户提供的照片路径（如有）

**简历结构设计：**

```
┌─ Header（姓名+照片+联系方式+求职意向）
├─ Tagline（一句话竞争力）
├─ 工作经历（按时间倒序，补充段标注[fabricated]）
├─ 项目经历（按与求职方向相关度排序）
├─ 教育背景
├─ 核心技能（表格化，含水平评级+证据来源）
├─ 自我评价（基于真实能力的差异化定位）
└─ 职业规划（短期/中期/长期）
```

**话术转化规则（参考简历优化提示词）：**

| 原始描述 | 转化后 |
|----------|--------|
| "我会用ChatGPT写文案" | "用AI搭建自动化内容管道，效率提升4倍" |
| "我会vibe coding" | "独立使用AI编程工具完成X项目，单文件N行代码" |
| "我会分配AI工作" | "设计M个Agent的并行工作流，覆盖X个业务环节" |

**输出格式 1: Markdown**
- 路径：`resume-versions/简历-{岗位方向}-{版本号}.md`
- 包含完整的元数据（frontmatter）
- `[fabricated]` 标记仅在源文件中，不出现在最终输出

**输出格式 2: HTML（含证件照）**
- 路径：`resume-versions/简历-{岗位方向}.html`
- 照片转 base64 data URI 内嵌
- 暗色 teal 主题（#0D9488），A4 打印友好
- `@media print` 样式配置
- 零外部依赖，双击即可打开

**输出格式 3: TXT（平台粘贴）**
- 路径：`resume-versions/简历-{岗位方向}.txt`
- 用 `=` 和 `─` 做视觉分隔
- 技能表格用空格对齐
- 支持直接复制粘贴到 BOSS直聘、微信等纯文本平台

---

### Stage 6: 质量审查（Quality Check）

最终输出前的自检清单：

- [ ] 所有技能描述都能追溯到 Stage 1 的具体项目
- [ ] 补充经历已标注 `[fabricated]` 并经用户确认
- [ ] 参考公司信息有可验证来源（URL/案号）
- [ ] 量化数据前后一致（不自相矛盾）
- [ ] HTML 版本照片正确内嵌（base64 非空）
- [ ] TXT 版本在纯文本环境下对齐正常
- [ ] 关键词密度匹配 Stage 3 推荐的岗位关键词
- [ ] 无敏感信息泄露（API Key、密码等）

## 案例目录结构

├── discovery-report.md        # Stage 1: 项目扫描报告

## 常见变体

### 变体 A：纯真实版（不补充经历）

适用于：有足够实习/项目经历的用户

跳过 Stage 4，所有工作经历来自 Stage 2 已有简历 + Stage 1 项目挖掘。

### 变体 B：多方向版

适用于：同时投递多个方向的用户

对每个目标方向重复 Stage 3-5，生成 `简历-方向A.html`、`简历-方向B.html` 等。

### 变体 C：快速迭代版

适用于：已有基础简历，只需针对特定 JD 优化

跳过 Stage 1-2，直接从已有简历 + 新 JD 出发，走 Stage 3→5→6。

## 运行示例

```
/resume-forge 求职方向：AI内容运营  目标城市：深圳/重庆  照片：C:\Users\xxx\Pictures\photo.jpg
```

系统将：
1. 自动扫描本地项目 → 输出能力矩阵
2. 搜索已有简历 → 提取可复用经历
3. 推荐 AI 内容运营相关岗位 → 三梯队排序
4. 搜索参考企业 → 提供工作经历素材（等用户确认）
5. 锻造三格式简历 → MD + HTML（含照片）+ TXT
6. 质量自检 → 输出最终简历

预计耗时：5-10 分钟（取决于项目数量和 Agent 响应速度）。
