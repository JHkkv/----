# AI任务收件箱 -- 产品需求文档（PRD 简版）

---

## 1. 产品一句话描述

**把你脑子里的乱七八糟，变成清清楚楚的待办清单。** 粘贴聊天记录、邮件片段、想法碎片，AI 自动帮你分类、排优先级、生成结构化任务卡片。

---

## 2. 用户画像

| 画像 | 身份 | 典型场景 | 核心痛点 |
|------|------|----------|----------|
| **碎片收集者** | 行政 / 运营 / 助理 | 微信群里被 @ 了 8 件事，邮件又来了 5 件，脑子记不住 | 信息散落在 5 个渠道，靠脑子记必然遗漏 |
| **多项目打工人** | 产品经理 / 项目经理 | 同时跟 4 个项目，每个都有不同截止日期和 stakeholders | 任务之间优先级靠感觉而不是系统判断 |
| **创意脑暴者** | 设计师 / 内容创作者 | 灵感随时冒出来，抓到就怕飞走 | 碎片想法没有快速捕获入口，等坐下来整理已经忘了一半 |
| **职场新人** | 应届生 / 实习生 | 被分配任务但不知道先做哪个，不好意思老问 mentor | 缺乏任务优先级判断框架，抓不住重点 |
| **自我管理者** | 自由职业 / 远程办公 | 工作与生活边界模糊，学习任务和工作任务混在一起 | 需要明确的分类系统来切换上下文 |

## 3. 用户核心流程

```
┌─────────────────────────────────────────────────────────────────┐
│  第1步：打开应用（PWA桌面图标或浏览器）                          │
│  ├─ 首次使用：看到空状态引导页 "粘贴你的第一条杂事，我来帮你整理" │
│  └─ 老用户：看到按优先级排列的任务卡片列表                        │
├─────────────────────────────────────────────────────────────────┤
│  第2步：粘贴/输入杂乱信息                                         │
│  ├─ 场景A：复制微信聊天记录 → 粘贴到输入框                        │
│  ├─ 场景B：手动打字 "明天前把Q3报告发给张总"                     │
│  └─ 场景C：粘贴一段长邮件正文                                    │
├─────────────────────────────────────────────────────────────────┤
│  第3步：AI自动处理（约1-3秒）                                     │
│  ├─ 拆解：如果一段文本包含多个任务，自动拆成多条                   │
│  ├─ 分类：归入预设分类（工作/个人/学习/其他 或用户自定义）        │
│  ├─ 优先级：自动标注 🔴加急/🟠重要/🟡普通/🔵低优                 │
│  └─ 时间戳：自动记录创建时间                                      │
├─────────────────────────────────────────────────────────────────┤
│  第4步：用户确认 & 微调                                           │
│  ├─ 检查 AI 分类对不对 → 不对就手动改                            │
│  ├─ 调优先级色标 → 点击切换四级颜色                               │
│  ├─ 修改时间 → 如果需要编辑创建时间或加截止时间                    │
│  └─ 补备注 → 在任务卡片上追加文字                                 │
├─────────────────────────────────────────────────────────────────┤
│  第5步：执行 & 跟踪                                               │
│  ├─ 看板视图：加急的自动排最前、同级按时间排                      │
│  ├─ 拖拽调整：手动把某张卡片拖到更靠前的位置                      │
│  ├─ 完成标记：勾选完成任务（移到已完成区/划掉）                    │
│  └─ 板块锁定：调整好布局后锁定，防止误触                           │
├─────────────────────────────────────────────────────────────────┤
│  第6步：回顾 & 清理                                               │
│  ├─ 查看历史完成记录                                              │
│  ├─ 清空已完成任务                                                │
│  └─ 导出数据（后续迭代功能）                                       │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 功能清单

### P0（MVP 必须上线）

| 功能 | 说明 |
|------|------|
| 文本输入框 | 支持粘贴多行文本，支持 Ctrl+Enter 快捷提交 |
| AI 智能拆解 | 一段文本含多个任务时自动拆成多条 |
| AI 自动分类 | 将任务归入预设分类（默认4类，用户可自定义） |
| AI 优先级判断 | 自动标注 🔴🟠🟡🔵 四级优先级 |
| 时间戳 | 自动生成创建时间，显示在卡片上 |
| 默认分类预设 | 工作 / 个人 / 学习 / 其他（至少 6 个扩展分类） |
| 列表展示 | 卡片式任务列表，加急排最前，同级按时间排序 |
| 手动优先级调整 | 点击颜色标签切换优先级 |
| 手动分类修改 | 下拉选择修改任务分类 |
| 任务完成 / 删除 | 勾选完成任务 & 删除功能 |
| localStorage 持久化 | 所有任务数据本地存储 |
| 空状态引导 | 首次使用展示引导文案 |

### P1（上线后第一轮迭代）

| 功能 | 说明 |
|------|------|
| 拖拽排序 | 板块可拖拽调整位置 |
| 布局锁定 / 解锁 | 锁定后禁止拖拽，防止误触 |
| 自定义分类 | 用户新增、编辑、删除分类 |
| 手动修改时间 | 编辑任务的创建时间 |
| PWA 支持 | 可安装到手机桌面，有离线缓存 |
| 离线可用 | Service Worker 缓存静态资源，无网也能查看已有任务 |
| AI 调用失败降级 | 网络异常时降级为纯文本输入 + 手动分类 |
| 超长文本处理 | 超过 2000 字自动截断并提示 |

### P2（后续规划）

| 功能 | 说明 |
|------|------|
| 批量导入 | 粘贴整段聊天记录，批量生成任务 |
| 搜索 & 筛选 | 按分类、优先级、关键词筛选 |
| 截止日期 | 为任务添加 deadline，到期提醒 |
| 数据导出 | 导出 JSON / CSV / Markdown |
| 多设备同步 | 云端同步（需要一个后端） |
| 主题切换 | 深色模式 / 浅色模式 |
| 统计面板 | 完成率、分类分布、周报生成 |

## 6. 默认分类预设

| 分类名 | 图标建议 | 适用场景 | 示例任务 |
|--------|----------|----------|----------|
| **工作** | 💼 | 本职工作任务、邮件、会议、报告 | "Q3报告发给张总"、"约会议室讨论方案" |
| **个人** | 🏠 | 生活琐事、家庭、健康、购物 | "周末去超市买日用品"、"预约牙科检查" |
| **学习** | 📚 | 课程、读书、技能提升、考证 | "完成 Figma 进阶课程第3节"、"读《设计心理学》第5章" |
| **紧急** | 🚨 | 突发情况、deadline 迫近的事项 | "客户投诉必须在2小时内回复" |
| **会议** | 📅 | 已安排的会议和需要准备的事项 | "周三10:00 项目评审会，准备演示PPT" |
| **灵感** | 💡 | 创意、想法、随手记 | "做个番茄钟 + 白噪音的小工具" |
| **待跟进** | 🔄 | 等待他人回复、阻塞中的任务 | "等李工确认接口对接时间" |
| **其他** | 📌 | 无法归入以上分类的杂项 | 兜底分类 |

> 首次使用时默认启用前 4 个（工作/个人/学习/其他），其余分类用户可在设置中开启或自定义。

## 7. AI 分类 Prompt 设计

### System Prompt

```
你是一个智能任务助手，负责将用户输入的杂乱信息拆解为结构化任务。

## 输入
用户会粘贴一段文本，可能是聊天记录、邮件片段、想法碎片、或混合内容。

## 任务
1. **拆解**：如果一段文本包含多个独立任务，拆成多条。每条任务必须是一个可执行的行动项（包含动词）。
2. **分类**：将每条任务归入以下分类之一：
   - 工作：与职业、公司、项目相关的任务
   - 个人：生活琐事、家庭、健康、购物
   - 学习：课程、读书、技能提升、考证
   - 紧急：需要立即处理、有明确紧迫 deadline 的事项
   - 会议：已安排或需要安排的会议
   - 灵感：创意、想法、brainstorm
   - 待跟进：等待他人回复、阻塞中的事项
   - 其他：无法归入以上分类的杂项

3. **优先级判断**：根据以下规则标注优先级：
   - urgent（加急）：明确包含"紧急"、"马上"、"立刻"、"今天下班前"、"deadline今天"等词，或涉及客户投诉/线上事故
   - important（重要）：包含明确的截止日期（明天/本周），或涉及关键 stakeholder（老板、客户）
   - normal（普通）：一般工作任务，没有紧迫时间要求
   - low（低优）：没有时间要求的想法、灵感、可以推迟的事项

4. **提取标题**：把每条任务提炼为不超过 30 字的简洁行动项。

## 输出格式
请严格输出 JSON 数组，不要包含任何其他文字：

[
  {
    "title": "简洁的行动项标题",
    "category": "工作",
    "priority": "important",
    "reasoning": "一句话说明分类和优先级判断依据"
  }
]

## 规则
- 如果输入不是任务（纯闲聊、无意义的字符），返回空数组 []
- 如果无法确定分类，默认归入"其他"
- 如果无法判断优先级，默认使用"normal"
- 每个 title 必须是一个可执行的动作，不能是模糊描述
- 不要编造用户没有提到的信息
- 如果输入只有1个任务，数组也只包含1个元素
```

## 8. 边界条件

### 8.1 空输入处理

| 场景 | 处理方式 |
|------|----------|
| 输入框为空直接提交 | 提示"请输入内容"，不发送 AI 请求 |
| 输入只有空格/换行 | 视为空输入，同上处理 |
| AI 返回空数组（非任务内容） | 提示"没有识别到可执行的任务，试试换个说法？" |
| AI 返回解析失败 | 降级为纯文本模式，由用户手动创建任务 |

### 8.2 超长文本处理

| 场景 | 处理方式 |
|------|----------|
| 输入超过 2000 字 | 截断到 2000 字并提示"内容过长，已截取前2000字，建议分段输入" |
| 单次 AI 返回超过 20 条任务 | 最多取前 20 条，超出部分提示"已生成20条任务，剩余内容请重新输入" |
| 单条任务标题超过 200 字 | 截断到 200 字 |

### 8.3 离线处理

| 场景 | 处理方式 |
|------|----------|
| 无网络时打开应用 | 正常显示已缓存的任务列表（PWA Service Worker 缓存） |
| 无网络时提交新任务 | AI 功能不可用，切换到手动词条模式：用户自行填写标题、选择分类和优先级 |
| 无网络时修改/完成任务 | 正常操作 localStorage，联网后无需同步（纯本地数据） |
| PWA 首次安装 | 缓存 App Shell（HTML/CSS/JS），显示安装成功提示 |

### 8.4 AI 调用失败降级

| 场景 | 处理方式 |
|------|----------|
| API 超时（>10秒） | 中止请求，提示"AI 响应超时，已切换为手动模式"，用户手动创建任务 |
| API 返回错误（4xx/5xx） | 提示"AI 服务暂不可用（错误码: xxx），已切换为手动模式" |
| API Key 未配置或过期 | 提示"AI 功能未配置，请在设置中填写 API Key，当前仅支持手动模式" |
| 网络波动（fetch failed） | 提示"网络异常，已切换为手动模式" |
| 连续失败 3 次 | 自动关闭 AI 功能，提示"AI 功能已暂时关闭，可随时在设置中重新开启" |
| 降级后的手动模式 | 显示完整表单：标题输入框 + 分类下拉 + 优先级选择 + 提交按钮 |

### 8.5 其他边界条件

| 场景 | 处理方式 |
|------|----------|
| localStorage 已满（5MB 上限） | 提示"存储空间不足，请清理已完成的任务后再添加" |
| 重复任务检测 | 如果新任务标题与已有未完成任务相似度 > 90%，提示"检测到可能重复的任务：[已有任务标题]，仍要添加吗？" |
| 任务拖拽到已完成区 | 自动标记为完成，并记录完成时间 |
| 删除分类 | 如果分类下还有未完成任务，提示"该分类下有 N 个任务，删除后这些任务将移至「其他」分类" |
