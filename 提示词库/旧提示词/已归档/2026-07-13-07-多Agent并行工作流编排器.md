# 多Agent并行工作流编排器

## 提示词内容

> 你是一个 AI Agent 工作流架构师。请将以下复杂任务拆解成多 Agent 并行执行的工作流：
>
> ## 任务描述
> [描述需要完成的复杂任务，如：搭建一个电商数据分析系统]
>
> ## 编排要求
> 1. **Agent 拆分**：将任务拆成 3-5 个独立 Agent，每个 Agent 有明确的角色和输入/输出
> 2. **并行设计**：标注哪些 Agent 可以并行运行，哪些有依赖关系
> 3. **数据流**：定义 Agent 之间的数据传递格式（输入什么 → 输出什么）
> 4. **聚合策略**：如何将多个 Agent 的输出合并为最终结果
> 5. **失败处理**：如果某个 Agent 失败或产出质量低，如何回退？
>
> ## 输出格式
> ```
> ## 工作流拓扑
> [Agent A] ──并行── [Agent B]
>     │                   │
>     └────→ [Agent C] ←──┘
>                │
>          [最终聚合]
>
> ## Agent 定义
> ### Agent A: [名称]
> - 角色：[描述]
> - 输入：[数据格式]
> - 输出：[数据格式]
> - 与其他 Agent 的关系：[并行/依赖]
>
> （依次定义所有 Agent）
>
> ## 数据流规范
> ## 失败回退策略
> ```
