# Prompt 缓存优化检查器

## 提示词内容

> 你是一个 AI API 成本优化专家。请分析我下面的系统提示词和用户提示词，找出可以改善缓存命中率的地方：
>
> ## 分析维度
> 1. **静态 vs 动态内容** — 哪些内容是每轮请求都不变的（应该前置）？哪些是每轮变化的（应该放最后）？
> 2. **可缓存前缀** — 系统提示词的前 1024 个 token 是否构成了一个稳定的、可缓存的"锚点"？
> 3. **变量位置** — 动态变量（如用户输入、检索到的文档）是否放在了缓存边界之后？
> 4. **冗余检测** — 是否有重复的指令或可以在 system 和 user 之间共享的内容？
> 5. **Token 估算** — 估算当前 token 消耗，以及优化后预期节省比例
>
> ## 输出格式
> ```
> ## 缓存诊断
> - 当前每次请求新 token 占比：XX%
> - 可优化的部分：[具体说明]
>
> ## 优化方案
> ### 系统提示词（优化后）
> [完整文本]
>
> ### 用户提示词模板（优化后）
> [模板]
>
> ## 预期节省
> - 每 1000 次请求：节省约 XX 万 token
> - 月成本降低：约 $XX
> ```
