# 主工程师级代码审查

## 提示词内容

> 以一位有不妥协标准的主工程师身份审查以下代码。检查三个维度：
> - **效率**：冗余 API 调用、未缓存的资源、资源泄漏
> - **韧性**：静默失败、缺失的错误边界、未处理的异常
> - **架构**：紧耦合、关注点分离不良、违反单一职责
>
> 对每个问题，说明：1) 可能导致什么生产故障？2) 精确的 git diff 修复方案。
