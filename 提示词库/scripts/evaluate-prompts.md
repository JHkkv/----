# 提示词评估 Agent 指令

> 此文件为定期评估 Agent 的执行指令。由 CronCreate 定时触发（每 3 天）。

## 任务

评估 `旧提示词/待评估/` 中所有状态为 pending 的提示词，决定激活/归档/保留。

## 评估维度（5 分制）

| 维度 | 权重 | 说明 |
|------|------|------|
| 可用性 (usability) | 30% | 拿来即用程度，是否需要大量修改 |
| 复杂度 (complexity) | 20% | 越简单越好（反向计分：1=最简单=5分，4=最复杂=2分） |
| 通用性 (generality) | 20% | 适用场景广度 |
| 新鲜度 (freshness) | 15% | 是否有新意，还是老生常谈 |
| 可靠度 (reliability) | 15% | 来源是否可信，是否有实际验证 |

## 决策规则

| 综合得分 | 操作 |
|----------|------|
| ≥ 3.5 | **激活** → 移入 `新提示词/{分类}/`，状态改为 active |
| 2.5 - 3.5 | **保留** → 留在待评估，等下轮 |
| < 2.5 | **归档** → 移入 `旧提示词/已归档/`，状态改为 archived |

## 执行步骤

1. 读取 `旧提示词/待评估/` 中所有 pending 文件
2. 逐条评分，写入评分字段（usability / generality / freshness / reliability）
3. 计算加权总分
4. 根据决策规则执行迁移：
   - 激活：复制到 `新提示词/{分类}/`，frontmatter 改为 active，删除待评估源文件
   - 归档：复制到 `旧提示词/已归档/`，frontmatter 改为 archived，删除待评估源文件
   - 保留：原地更新评分，不移动
5. 写入优化记录：`f:/测试工具/提示词库/优化记录/YYYY-MM-DD.md`

## 优化记录格式

```markdown

---

| 保留 | N |

## 评估详情
### 激活项
| 文件 | 评分 | 目标分类 |
|------|------|----------|

### 归档项
| 文件 | 评分 | 原因 |
|------|------||
```

## 执行完毕后

1. 更新 `f:/测试工具/提示词库/索引.md` 的统计数字和新提示词列表
2. 如有激活项，同步更新关系网（检查 related 字段，建立关联）
