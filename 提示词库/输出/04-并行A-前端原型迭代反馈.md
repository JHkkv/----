## 二、P0 Hotfix（本周必须修复）

### 2.1 移动端时间选择器嵌套滚动冲突

**问题描述**：DatePickerPopover 在移动端（320px-375px 视口）展开时，时间滚轮与页面主滚动区域存在事件冲突，导致用户无法精确选择时间。

**复现路径**：移动端 → 桌面端点击日历 → DatePickerPopover 弹出 → 尝试滚动选择时间

**根因**：Popover 内部使用原生 scroll 事件，未对 touch 事件做 `stopPropagation`，外层可滚动容器捕获了 touchmove。

**修复方案**：
```tsx
// DatePickerPopover.tsx
// 在 PopoverContent 上添加触摸事件隔离
<PopoverContent
  className="..."
  onTouchMove={(e) => e.stopPropagation()}  // 阻断外层滚动
  onWheel={(e) => e.stopPropagation()}
>
```

**验收标准**：移动端可流畅选择时间，无页面跳动

### 2.2 热力图 Tooltip 被相邻单元格裁切

**问题描述**：热力图中 Tooltip 内容（包含日期、总时长、各应用耗时列表）在单元格密集区域被 `overflow: hidden` 裁切，用户无法看到完整信息。

**复现路径**：首页热力图 → 鼠标悬停在非边缘单元格 → Tooltip 内容被截断

**根因**：热力图容器设置了 `overflow: hidden` 用于边界控制，Tooltip 作为子元素被包含在内。

**修复方案**：将 Tooltip 改为 Portal 渲染，脱离热力图容器的 overflow 约束：
```tsx
// HeatmapChart.tsx
import { createPortal } from 'react-dom'

// Tooltip 使用 Portal 渲染到 body
const tooltipContent = (
  <div className="fixed z-50 ..." style={{ left: x, top: y }}>
    {/* Tooltip 内容 */}
  </div>
)

return createPortal(tooltipContent, document.body)
```

**验收标准**：Tooltip 在任意位置均可完整显示，不被裁切

---

### 2.3 排行榜中位数对比线缺失

**问题描述**：PRD 要求排行榜柱状图需显示"全用户中位数"对比线，当前原型中该元素完全缺失，用户无法判断自己的排名处于什么水平。

**涉及页面**：排行榜（桌面端 + 移动端）

**修复方案**：在 Recharts BarChart 中添加 ReferenceLine：
```tsx
<BarChart data={rankData}>
  <Bar dataKey="hours" fill="var(--accent)" />
  <ReferenceLine
    y={medianHours}
    stroke="var(--color-muted)"
    strokeDasharray="4 4"
    label={{
      value: `中位数 ${medianHours}h`,
      position: 'right',
      fill: 'var(--color-muted)',
      fontSize: 12,
    }}
  />
</BarChart>
```

**验收标准**：排行柱状图中可见虚线中位数对比线，标注文字清晰

## 三、P1 周迭代优化（V2 Sprint 内完成）

### 3.1 情绪模式状态栏感知优化

**当前状态**：底部状态栏的模式切换按钮缺乏视觉层次，闹钟模式图标辨识度不足。

**优化方案**：
- 番茄钟模式：钟表图标 + 运行中脉冲动画（CSS `@keyframes pulse`，1.5s 无限循环）
- 闹钟模式：闹钟图标增加 `ring` 动画效果（铃铛摇摆，0.3s，ease-out）
- 模式切换时添加交叉淡入淡出（`opacity` 过渡 200ms）
- 桌面端 Tooltip 增加快捷键提示（如"切换模式 · Space"）

### 3.2 历史趋势视图切换动效

**当前状态**：日/周/月切换为瞬时替换，无过渡反馈，用户容易迷失上下文。

**优化方案**：
- 使用 CSS `view-transition-name` 实现平滑切换（Chromium 111+ 支持）
- 降级方案：内容区域使用 `opacity` + `translateY` 的 300ms 过渡
```css
.trend-view-enter {
  opacity: 0;
  transform: translateY(8px);
}
.trend-view-active {
  opacity: 1;
  transform: translateY(0);
  transition: all 300ms var(--ease-out-expo);
}
```

### 3.3 Tab 切换动画增强

**当前状态**：导航 Tab（概览/详情/目标）切换时仅有颜色渐变，缺乏空间感。

**优化方案**：
- 添加下划线滑动指示器（`transform: translateX()`，250ms）
- Tab 内容区域使用 `crossfade` 过渡
- 移动端 Tab 点击增加 `ripple` 反馈

### 3.4 热力图悬停状态增强

**当前状态**：单元格悬停仅有简单的背景色变化，与周围单元格的对比不够明显。

**优化方案**：
- 悬停单元格增加 `transform: scale(1.1)` 微放大
- 同时降低周围 8 格的 opacity 至 0.5，聚焦当前单元格
- 添加 `box-shadow` 提升层次感
```css
.heatmap-cell:hover {
  transform: scale(1.1);
  box-shadow: 0 0 0 2px var(--accent);
  z-index: 10;
}
```

### 3.5 目标进度环完成动画

**当前状态**：目标完成时进度环无庆祝反馈，缺乏成就感。

**优化方案**：
- 进度达到 100% 时播放 SVG stroke-dasharray 动画（环形填充 0→360°，1s）
- 触发 confetti 微粒子效果（使用 `canvas-confetti` 库，单次触发，30 粒子）
- 目标卡片添加绿色发光边框（`box-shadow: 0 0 20px rgba(16, 185, 129, 0.3)`）

### 3.6 日期范围快捷选择器

**当前状态**：排行榜的"本周/上周/本月"切换使用普通按钮组，缺乏时间语义。

**优化方案**：
- 改为 SegmentedControl 样式，带滑动背景指示器
- 选中项显示对应日期范围的缩略标注（如"6.22 - 6.28"）
- 切换时图表数据使用 `animate` 属性平滑过渡

### 3.7 柱状图堆叠动画

**当前状态**：排行榜柱状图为静态渲染，首次加载无数据进入效果。

**优化方案**：
- 首次渲染使用 Recharts 内置 `animationBegin` 和 `animationDuration`
- 设置 `animationEasing="ease-out"`，`animationDuration={800}`
- 各柱体交错 50ms 开始动画（`animationBegin={index * 50}`）

### 3.8 移动端底部导航视觉优化

**当前状态**：移动端底部 Tab 栏高度偏小（44px），图标与文字间距不足。

**优化方案**：
- 底部导航栏高度提升至 56px
- 图标与文字间距增加至 4px
- 当前激活 Tab 增加底部圆点指示器（4px 圆点，accent 色）
- 安全区域适配（`padding-bottom: env(safe-area-inset-bottom)`）

## 四、P2 体验优化（V2+ 延后）

### 4.1 心情标记 UI 重构

**当前状态**：心情标记使用 Emoji 选择器，与整体深色科技风格不够协调。

**优化方案**：
- 改为 5 级 Likert 量表样式（极差/差/一般/好/极好）
- 使用几何图标替代 Emoji（三角形→圆形→五角星递进）
- 选中态使用 accent 色填充 + 微弹跳动画

### 4.2 视觉风格深化

**当前状态**：已遵循深色科技风，但部分组件（如卡片、按钮）仍使用默认阴影和圆角，缺乏品牌感。

**优化方案**：
- 统一卡片 `border: 1px solid rgba(255,255,255,0.06)`（参考风格指南 glassmorphism）
- 按钮增加 `backdrop-filter: blur(8px)` 毛玻璃效果
- 添加微妙的网格纹理背景（CSS `background-image` 0.5px 线条，opacity 0.03）
- 数字使用 Tabular Lining 确保对齐

### 4.3 图表色彩统一

**当前状态**：各图表的颜色映射不完全一致（热力图、趋势图、排行榜的应用颜色有偏差）。

**优化方案**：
- 抽取应用颜色映射为全局常量 `APP_COLOR_MAP`
- 所有图表组件从同一数据源读取颜色
- 新增应用自动分配颜色（基于 hue 环均分，避免与已有颜色冲突）
