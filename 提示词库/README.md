# 🧠 智能提示词管理系统

## 概述

这是一个自动化 AI 提示词管理系统，包含三个核心 Agent：

| Agent | 功能 | 触发方式 |
|-------|------|----------|
| **搜索 Agent** | 每日自动搜集 AI 提示词 | CronCreate 每日 10:00 |
| **判断 Agent** | 评估、分类、优化提示词 | CronCreate 每 3 天 + 搜索后自动 |
| **运用 Agent** | 根据任务匹配提示词 | `/prompt-use` Skill |

## 目录结构

```
提示词库/
├── 新提示词/         ✅ 活跃可用（按分类）
├── 旧提示词/         📦 归档/待评估
├── 关系网/           🔗 提示词关联
├── 模板/             📝 新建模板
├── 原始搜集/         🔍 每日搜索结果
├── 优化记录/         📊 优化日志
└── 索引.md           📚 全库总览
```

## 使用方法

### 1. 搜索提示词（自动）

系统每日自动搜索，结果写入 `原始搜集/` 和 `旧提示词/待评估/`。

### 2. 评估提示词（自动）

- 搜索后自动评估
- 每 3 天运行优化周期
- 高分 → `新提示词/`，低分 → `旧提示词/已归档/`

### 3. 使用提示词

在 Claude Code 中输入：

```
/prompt-use 帮我写一篇产品文案
```

系统会：
1. 从 `新提示词/` 匹配最佳提示词
2. 返回 Top 3 候选
3. 组合生成最终 Prompt
4. 记录使用反馈

### 4. 在 Obsidian 中浏览

- 打开 Obsidian → 打开 `f:\测试工具\提示词库` 作为 vault
- 使用 Graph View 查看提示词关系网络
- 通过标签和链接浏览相关提示词

## 评估标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 可用性 | 30% | 直接可用程度 |
| 复杂度 | 20% | 简单明了程度（反向） |
| 通用性 | 20% | 适用范围广度 |
| 新鲜度 | 15% | 新颖程度 |
| 可靠度 | 15% | 来源可信度 |

**总分 ≥ 3.5** → 新提示词（活跃）
**2.5 - 3.5** → 待评估
**< 2.5** → 已归档

## 关系网

提示词之间通过 `related` 字段建立关联，在 Obsidian 的 Graph View 中可视化呈现。关联类型：

- **相似**：功能相近的提示词
- **互补**：配合使用效果更好
- **升级**：旧版本 → 新版本
- **组合**：可合并为更强大的提示词
