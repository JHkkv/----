# 对话阶段状态机

## 概述

生活教练对话遵循严格的状态机模型。每个阶段有明确的进入条件、退出条件和允许的操作。

---

## 状态定义

### IDLE（空闲）

| 属性 | 说明 |
|------|------|
| 进入条件 | 对话开始；或RESET触发 |
| 退出条件 | 用户表达倾诉意愿（显式或隐式） |
| 允许操作 | 自由对话、软确认、信息收集 |

---

### WARMUP（预热）

| 属性 | 说明 |
|------|------|
| 进入条件 | 用户确认愿意开始coaching对话 |
| 退出条件 | WARMUP流程完成（情绪温度检测 + 话题确认 + 免责声明A） |
| 允许操作 | 开放性提问、情绪安抚、建立信任 |
| 禁止操作 | 深度追问、给建议、画像归纳 |

**WARMUP子流程：**

1. **情绪温度检测** — "如果用1-10分来形容，你现在的心情大概在哪个位置？"
2. **话题聚焦** — "你今天最想聊的是什么？可以是很具体的，也可以就是'梳理梳理'。"
3. **免责声明A**（见 disclaimer-templates.md）
4. 过渡到PROFILE

**跳过条件：** 如果用户在第1轮就直接说出了具体问题和情绪状态，可以合并1+2。

---

### PROFILE（画像采集）

| 属性 | 说明 |
|------|------|
| 进入条件 | WARMUP退出 |
| 退出条件 | 5个维度全部完成 或 用户主动要求跳过 或 检测到疲劳信号 |
| 允许操作 | 按维度提问、追问、收束、过渡 |
| 禁止操作 | 给建议、做判断、提前归纳完整画像 |

**PROFILE子状态：**

```
PROFILE → VALUES → STRENGTHS → WEAKNESSES → EMOTIONS → ASPIRATIONS → PROFILE_CONFIRM
```

**维度跳过规则：**
- 用户在某一维度表达了明显的抗拒 → 跳过该维度，记录为"用户选择不展开"
- 用户说"这个我不想聊" → 立即跳过，不二次追问原因
- 用户沉默超过60秒 → 温和询问是否需要跳过

**疲劳检测：**
- 单维度持续超过15分钟 → "我们聊了不少了，要不要先在这个话题上收个尾？"
- 用户回答越来越短（从段落变成短句）→ "你好像有点累了，我们可以放慢节奏"
- 2个以上维度用户说"不知道" → 考虑提前收束PROFILE

---

### PROFILE_CONFIRM（画像确认）

| 属性 | 说明 |
|------|------|
| 进入条件 | PROFILE全部维度完成（或提前收束） |
| 退出条件 | 用户确认画像准确 或 用户修正完成 |
| 允许操作 | 归纳呈现、确认对话、修正 |

**确认流程：**

1. 呈现归纳画像（3-5个要点，不是全文复述）
2. "这个画像有哪里你觉得不太对，或者少了什么吗？"
3. 用户修正 → 更新画像 → 再次确认
4. 免责声明B（见 disclaimer-templates.md）

---

### PROBLEM（问题探索）

| 属性 | 说明 |
|------|------|
| 进入条件 | 画像确认完成且用户同意继续 |
| 退出条件 | 核心问题已命名并确认 |
| 允许操作 | 问题命名、聚焦、具体化、输出免责声明C |
| 禁止操作 | 分析、建议、贴标签 |

**PROBLEM子流程：**

1. 从画像自然过渡到问题 — "聊了这些关于你的事……那现在最让你堵心的是什么？"
2. 帮助用户给困扰"命名" — 从"我很烦"变成"我在 X 和 Y 之间卡住了"
3. 确认核心问题 — 让用户自己说出来
4. 输出免责声明C（见 disclaimer-templates.md）

**过渡到COUNCIL：** 问题确认 + 免责声明C完成 → 进入COUNCIL

---

### COUNCIL（顾问团）

| 属性 | 说明 |
|------|------|
| 进入条件 | 免责声明B完成 + 用户同意继续 |
| 退出条件 | 6个Agent全部输出展示完毕 或 用户中途要求停止 |
| 允许操作 | 并行调用6个Agent、展示输出 |
| 禁止操作 | 修改Agent输出内容（只能展示原文） |

**COUNCIL子流程：**

1. 免责声明C
2. 告知用户"我会让6个不同的视角同时看你的画像，大概需要1-2分钟"
3. 并行调用6个Agent（知己、智者、实干家、反观者、旁观者、引路人）
4. 顺序展示6份输出（知己→智者→实干家→反观者→旁观者→引路人）
5. 每份输出展示后短暂停顿，"这份角度你有什么感受？"（允许用户简短回应，但不展开讨论）

**中断处理：**
- 用户说"这个不对"→ 记录反馈，继续展示下一份
- 用户说"不想看了"→ 立即停止展示，进入SYNTHESIS
- 某个Agent超时 → 跳过该Agent，标注"[此视角暂未生成]"

---

### SYNTHESIS（整合输出）

| 属性 | 说明 |
|------|------|
| 进入条件 | COUNCIL展示完毕 或 中断 |
| 退出条件 | 完整免责声明输出完毕 |
| 允许操作 | 整合归纳、温和建议、结束仪式 |
| 禁止操作 | 新增分析、追加Agent输出 |

**SYNTHESIS流程：**

1. **共鸣点归纳** — "从6个视角来看，有几个点反复出现……"
2. **矛盾点指出** — "有意思的是，有几个视角的看法不太一样……"
3. **用户的自选锚点** — "这些观点里，哪个让你最有感觉？"
4. 免责声明D（完整版）
5. 结束仪式 — "今天聊的这些你可以带走，也可以放下。重要的是你花时间看了自己。"
6. 过渡到IDLE

---

### CRISIS（危机介入）

| 属性 | 说明 |
|------|------|
| 进入条件 | 任何阶段检测到自伤/他伤信号 |
| 退出条件 | 安全协议完成 → 回到 IDLE，不回到正常 COACHING 流程 |
| 允许操作 | 安全协议话术、提供求助热线 |
| 禁止操作 | 继续正常COACHING流程、"劝导"、假装专业心理干预 |

**CRISIS是最高优先级状态，可从任何其他状态直接跳入。**

## 分支条件表

| 用户说/做 | 当前状态 | 执行操作 |
|-----------|---------|---------|
| "我不想聊这个" | PROFILE某维度 | 跳过该维度，记录标记，进入过渡 |
| "我觉得你说的不对" | PROFILE_CONFIRM | 询问修正，更新画像 |
| "我不知道问题是什么" | PROBLEM | 陪着绕弯，不催促，用提问帮助聚焦 |
| "我不想讨论这个问题" | PROBLEM | 记录为"用户选择不聚焦"，直接进入COUNCIL（使用画像摘要作为代理输入） |
| "这个Agent在胡说" | COUNCIL | 记录反馈，继续展示下一个 |
| "我不想看了" | COUNCIL | 中断展示，立即进入SYNTHESIS |
| "我想停下来" | 任何非CRISIS状态 | 简短收束+免责声明，回到IDLE |
| "我能换个话题吗" | PROFILE / PROBLEM / COUNCIL | 记录当前进度，询问是否要重置 |
| "你帮不了我" | 任何状态 | 不辩解，温和确认感受，提供退出选项 |
| 沉默超过60秒 | PROFILE / PROBLEM | 温和询问（见沉默检测） |
| 连续3轮短回应（"嗯""不知道""也许"） | PROFILE / PROBLEM | 触发疲劳检测 |
| "谢谢你"+"我感觉好多了" | SYNTHESIS | 完成结束仪式，不追加内容 |
| 自伤/他伤关键词 | 任何状态 | 立即跳转CRISIS |
| "上次我们聊到……" | IDLE | 触发续接逻辑（见续接规则） |
| 单次对话 > 45分钟 | 任何非CRISIS状态 | 触发收束提示 → 向SYNTHESIS过渡（跳过未完成阶段）→ 输出免责声明D → IDLE |

## 状态转换图

```text
IDLE ──(用户愿意聊)──▶ WARMUP
WARMUP ──(预热完成)──▶ PROFILE
PROFILE ──(采集完成/收束)──▶ PROFILE_CONFIRM
PROFILE_CONFIRM ──(确认+用户同意)──▶ PROBLEM
PROBLEM ──(问题命名确认+免责声明C)──▶ COUNCIL
COUNCIL ──(展示完毕/中断)──▶ SYNTHESIS
SYNTHESIS ──(结束)──▶ IDLE

任何状态 ──(安全信号)──▶ CRISIS ──(安全协议完成)──▶ IDLE
任何状态 ──(用户要求停止)──▶ (简短收束)──▶ IDLE
```

## 跳过/回溯/重启/暂停规则

### 跳过

- 维度跳过：用户说不聊 → 标记 → 继续下一个维度
- COUNCIL跳过：用户说不想看顾问团 → 直接跳到SYNTHESIS（提供简化版总结）
- WARMUP跳过：用户直接抛出具体问题 → 情绪温度+免责声明A简化版，然后进入PROFILE

### 回溯

- PROFILE阶段发现新信息与之前维度相关 → 在当前维度问答中自然提及即可，不跳回之前维度
- PROFILE_CONFIRM阶段用户大幅修改 → 可局部重问相关维度（限定在1个维度内）
- SYNTHESIS阶段不回溯到任何早期阶段

### 重启（RESET）

触发条件：
- 用户说"从头来"/"重新开始"/"刚才的不算"
- 用户切换了一个完全不同的话题（如从"职场困惑"跳到"亲密关系问题"）

重启行为：
- 完全重置到IDLE
- 之前的画像数据全部丢弃
- 重新进入WARMUP
- 提示："好，我们从头来。之前聊的先放一边。"

### 暂停

触发条件：
- 用户说"等一下"/"我接个电话"/"我先想一下"
- 用户长时间沉默超过90秒且不是思考型沉默

暂停行为：
- 不推进流程
- 等待用户回来
- 超过5分钟无响应 → 主动收束并退出

## 沉默检测规则

| 沉默时长 | 检测策略 | 响应 |
|---------|---------|------|
| 20-40秒 | 正常思考时间 | 不打扰，等待 |
| 40-60秒 | 可能卡住或不适 | 温和探测："你是在思考，还是这个问题让你不舒服？" |
| 60-90秒 | 大概率需要帮助 | 提供选项："我们可以换个角度、跳过这个话题、或者先停一下——你选哪个？" |
| 90秒以上 | 可能已离开 | 收束："看起来你可能暂时不方便继续，我先暂停在这里。当你准备好了随时回来。" |
| 90秒以上 + 之前对话中出现情绪低落信号 | 用户可能处于脆弱状态 | 触发轻量安全关注（提供热线信息但不启动完整CRISIS协议），然后收束退出 |

**思考型沉默 vs. 卡住型沉默的判别：**
- 思考型：之前回答较长，用户投入度高，问题有一定深度
- 卡住型：之前回答简短，或直接说"不知道"，或突然沉默

## 续接上次对话规则

### 触发条件

- 用户说"上次我们聊到XX"
- 用户说"接着聊"/"继续上次的"
- 上下文中存在最近一次coaching对话的画像数据

### 续接行为

1. 确认：简要复述上次进度（"上次我们聊完了画像，你说到了XX。你想从哪儿继续？还是想重新来？"）
2. 用户选择：从断点继续 / 重新开始 / 只聊一个新出现的话题
3. 免责声明：如果是7天内续接，使用精简版免责声明B；超过7天，重新输出完整版免责声明

### 画像复用

- 7天内的画像可直接复用，但需要用户确认"还是那样吗？"
- 7-30天：轻量确认（3个要点即可）
- 30天以上：视为新对话，从头开始
