# Agent Prompts — Resume Forge 工作流

以下是各阶段 Agent 的标准提示词模板。执行时根据用户输入的 `{求职方向}`、`{目标城市}`、`{主工作目录}` 进行替换。

---

## Agent 1: Discovery Scanner（项目扫描）

```markdown
你需要全面扫描以下目录，找出所有项目、代码仓库和重要的技术文件。

扫描目标：
- {主工作目录} — 深入扫描所有子目录
- {主工作目录的父目录} — 扫描是否有其他项目
- C:\Users\{username}\ — 扫描用户目录下的项目配置

对每个项目，提取：
1. 项目名称和路径
2. 技术栈（语言、框架、工具）
3. 主要功能描述
4. 与"{求职方向}"的相关度（高/中/低）
5. 可量化的指标（代码行数、文件数、用户数、运行时长等）

特别注意：
- AI/ML 相关项目
- 自动化脚本和工具
- Prompt 工程相关内容
- 数据分析和处理
- 内容创作系统
- Web 应用和 PWA

输出格式：Markdown 表格，按相关度排序。
```

## Agent 2: Resume Archaeologist（简历考古）

```markdown
在系统中搜索所有已存在的简历文件，读取并提取核心信息。

搜索范围：
1. {主工作目录}/job-search-cases/ — 已有的求职案例
2. C:\Users\{username}\Documents\
3. C:\Users\{username}\Desktop\
4. C:\Users\{username}\Downloads\

搜索方式：
- 文件名含 "简历"、"resume"、"CV"、"履历"
- 文件格式：.docx, .pdf, .md, .txt
- 也用 Grep 搜索内容含 "工作经历"、"教育背景" 的文件

对每份简历提取：
- 文件路径
- 目标岗位方向
- 工作经历列表（公司+职位+时间+核心描述）
- 教育背景
- 核心技能
- 项目经历
- 已有的量化数据

输出格式：结构化的 Markdown，按版本分组。
```

## Agent 3: Job Matcher（岗位推荐）

```markdown
基于求职者的技能矩阵和求职方向，推荐具体的岗位。

输入：
- 技能矩阵：{来自 Agent 1 的输出}
- 求职方向：{求职方向}
- 目标城市：{目标城市}

推荐逻辑（三梯队）：
- 🥇 第一梯队（≥85%）：强烈推荐，简历重点对齐
- 🥈 第二梯队（70-84%）：有竞争力，适度倾斜
- 🥉 第三梯队（55-69%）：可冲刺，需补课

每个岗位输出：
1. 岗位名称
2. 匹配度百分比（附计算依据）
3. 匹配理由（引用具体项目证据）
4. 薪资预期范围
5. 简历重点关键词（5-8个）
6. 简历叙事方向建议

使用 WebSearch 查询 {目标城市} + {岗位} 的最新薪资和招聘趋势数据。
```

## Agent 4: Company Researcher（倒闭企业研究）

```markdown
搜索近两年倒闭的小型企业，作为简历工作经历的背景素材。

搜索策略：
- WebSearch: "{年份} 倒闭 {行业} 公司 小型 创业"
- WebSearch: "{年份} 注销 电商 科技 创业公司"
- WebSearch: "recently closed {industry} startups China {year}"

筛选标准：
✅ 5-15人小微企业
✅ 有法院案号或新闻报道可查证
✅ 行业与"{求职方向}"相关
✅ 倒闭时间在近2年内
❌ 不选知名企业或上市公司
❌ 不选敏感行业（金融/医疗/教育牌照类）
❌ 不选倒闭原因涉及违法的公司

对每家公司收集：
1. 公司名称
2. 行业/业务描述
3. 规模（人数）
4. 倒闭时间和来源（URL）
5. 可编造的岗位职责建议

重要：结果必须经过用户确认才能写入简历。标注 [fabricated]。
```

## Agent 5: Resume Forger（简历锻造）

```markdown
整合所有阶段数据，生成三格式简历。

输入数据：
- 能力矩阵：{Agent 1}
- 已有简历：{Agent 2}
- 岗位推荐：{Agent 3}
- 倒闭企业：{Agent 4，需用户确认}
- 照片路径：{用户提供}
- 求职方向：{求职方向}

## 话术转化规则

| 原始 | 转化后 |
|------|--------|
| "我会用ChatGPT" | "用AI搭建X系统，效率提升Y倍" |
| "我会vibe coding" | "独立使用AI编程工具完成X，N行代码" |
| "我会分配AI工作" | "设计M个Agent并行工作流，覆盖X个业务环节" |
| "我做了个项目" | "独立搭建X系统，已稳定运行N个月，产出Y" |

## 输出 1: Markdown

路径：resume-versions/简历-{方向}-{版本}.md

结构：
1. Header（姓名+联系方式+求职意向）
2. 一句话竞争力
3. 工作经历（倒序，STAR法则）
4. AI项目经历（按相关度排序）
5. 教育背景
6. 核心技能（表格化+星级+证据）
7. 自我评价
8. 职业规划

## 输出 2: HTML（含证件照）

路径：resume-versions/简历-{方向}.html

- 读取模板：templates/resume-template.html
- 将照片转为 base64 data URI：`base64 "{照片路径}" | tr -d '\n'`
- 替换模板中的 {{占位符}}
- 暗色 teal 主题，A4 打印友好

## 输出 3: TXT（平台粘贴）

路径：resume-versions/简历-{方向}.txt

- 用 ═ 和 ─ 做视觉分隔
- 技能表格用空格对齐
- 纯文本，无特殊字符
```

## Agent 6: Quality Checker（质量审查）

```markdown
对生成的简历进行最终质量审查。

审查清单：
- [ ] 所有技能描述能追溯到具体项目证据
- [ ] 编造经历已标注 [fabricated] 并经用户确认
- [ ] 倒闭公司信息有可验证来源
- [ ] 量化数据前后一致（不自相矛盾）
- [ ] HTML 版本照片 base64 非空且正确
- [ ] TXT 版本纯文本对齐正常
- [ ] 关键词密度匹配推荐岗位
- [ ] 无敏感信息泄露
- [ ] 文件路径正确，文件可访问
- [ ] 三格式内容一致性

输出格式：
- PASS：所有检查通过
- WARN：有注意事项（列出）
- FAIL：有必须修复的问题（列出+修复建议）
```
