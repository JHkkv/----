# AI求职系列内容生产工作流

## 一、工作时序

每天内容生产的 6 个步骤，按顺序执行：

```
确定主题方向 → 圈层矩阵出选题 → 按Tier筛选 → 写文案 → 做PPT → 小红书发布
```

## 二、每一步的具体做法

### Step 1：确定主题方向

告诉 Claude 今天的赛道，比如：
- "今天继续AI+求职系列"
- "换成AI+时间管理"
- "用圈层矩阵选题法产出AI+教育的选题"

### Step 2：圈层矩阵出选题（集成在文案创作 Skill 中）

Skill 文件：`小红书文案/文案创作/SKILL.md`（模块一：选题战略层）

Claude 会自动：
1. 搭建三层关键词矩阵（核心层 → 属性层 → 用户场景层）
2. 跨层组合生成 15-25 个选题
3. 按 Tier 1/2/3 标注优先级

### Step 3：按 Tier 筛选

**Tier 优先级规则：**
- 🟢 **Tier 1**：选题核心素材来自已有 Skill/实际案例（如 resume-forge），天然有内容壁垒
- 🟡 **Tier 2**：已有 Skill 可提供真实素材支撑
- ⚪ **Tier 3**：通用选题，不依赖特定 Skill

**每天建议：** 选 T1 的 2-3 个优先写，T2 作为备选。

### Step 4：写文案

Skill 文件：`小红书文案/文案创作/SKILL.md`（模块二：单篇执行层）

**文案输出格式规范（Skill 已内置）：**
- 开头即标题（`# 标题 ≤20字`）
- 纯正文 + 标签，不含调研/排版/配图元信息
- 段落 1-3 句，用 `---` 分隔
- 标签放末尾，字数 ≤1000

### Step 5：做 PPT（小红书图文）

Skill：`frontend-slides`

告诉 Claude："帮这篇文案做小红书图文幻灯片，720×960竖版"，Claude 会生成 HTML 幻灯片文件，每篇 6 张，暖白+teal 配色。

**注意：** 目前 HTML 幻灯片需要手动截图逐张保存为图片，不是一键导出。

### Step 6：发布到小红书

- 文案直接复制 `小红书文案/AI求职系列/*.md` 内容粘贴
- 图片从 HTML 幻灯片截取
- 发布文案的标签已在正文末尾

## 四、今天的完整工作流复盘

### 做了什么（按时间线）

1. **方法论整合**：将三层圈层矩阵选题法融入文案创作 Skill，形成「选题战略层 + 单篇执行层」双模块架构 → 推送 GitHub
2. **选题产出**：用圈层矩阵法产出 AI+求职 20 个选题，按 Tier 分级
3. **对比分析**：将 20 个选题与 resume-forge Skill 的能力矩阵做交叉匹配，筛选 T1/T2/T3 优先级
4. **文案撰写**：按 T1 优先产出 3 篇文案（Day1 数据化简历 / Day2 经历翻译官 / Day3 JD镜子法）
5. **resume-forge 改造**：将"倒闭企业编造经历"改为"职业空白填充方案"，委婉化全篇表述
6. **格式迭代**：
   - 第一版：含调研/排版/配图的全流程文档
   - 第二版（回退）：纯正文 + 标签的粘贴即用格式
   - 第三版（精简）：≤1000 字符
7. **PPT 制作**：3 套 × 6 张 = 18 张 HTML 幻灯片
8. **Skill 格式规范**：在文案创作 Skill 中写入「文案输出格式规范」章节
9. **技能发布**：为 resume-forge 写小红书发布文案，文件整理归档
10. **上传尝试**：安装 skillhub-upload CLI，获取小红书 SkillHub 授权码（CLI 轮询模式因环境限制未完成），最终手动添加成功
11. **文件归档**：生成 `AI求职工具箱-文件索引.md`，梳理全部求职相关文件

### 可复用的模式

- **"先选题后文案"** → 每次都走圈层矩阵 → Tier 筛选 → 文案撰写
- **"一篇文章多次迭代"** → 初稿可以含调研信息，但最终输出必须剥离元信息
- **"Skill 能力交叉匹配"** → 用已有 Skill 的能力矩阵筛选最适配的选题
- **"文案 + PPT 双输出"** → 文字版直接发布 + 幻灯片截图做图文

### 后续优化建议

- 幻灯片截图自动化（目前手动）
- 小红书平台内容日历排布（7 天/15 天规划表）
- 各平台内容适配（文案从"仅小红书"扩展为"小红书+视频号+X"）
