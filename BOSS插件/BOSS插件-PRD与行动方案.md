## 二、产品定位（修正后）

### 原始定位
> "Chrome扩展插件，自动生成招呼语+简历优化+自动搜索+一键投递"

### 修正后定位（经对抗审查）
> **独立PWA + 移动端优先的AI求职写作助手**
>
> - 用户在任何平台（Boss/51job/智联）看到心仪岗位
> - 打开PWA → 粘贴JD链接或截图 → AI生成个性化招呼语
> - 一键复制 → 切回Boss → 粘贴 → **手动点击发送**
> - **不触碰Boss的DOM，不违反ToS，不被封号**
> - **手机端可用（这才是主力使用场景）**

### 一句话价值主张
> "不是帮你海投的工具，而是帮你写出让HR点开的招呼语的AI参谋。"

## 三、MVP范围（10天极简版）

### MVP核心功能（只做一个）

**AI招呼语生成**

1. 用户粘贴岗位JD文本（或输入JD链接）
2. 用户录入个人背景（姓名、核心技能、关键经历数据）
3. AI分析JD → 提取关键词 → 匹配用户经历 → 生成3版本招呼语
4. 使用6种钩子策略（数字钩子/匹配钩子/成就钩子等）
5. 用户一键复制到剪贴板

### MVP不做清单

- ❌ 不做Chrome扩展（先用PWA验证）
- ❌ 不做DOM操控/页面注入
- ❌ 不做自动发送/一键投递
- ❌ 不做简历优化
- ❌ 不做自动搜索
- ❌ 不做多平台适配
- ❌ 不做远程配置热更新

### MVP验收标准

1. 在手机浏览器上打开PWA，粘贴JD文本 → AI生成招呼语 → 复制成功
2. AI生成时间 < 8秒（流式输出首字 < 2秒）
3. 3版本招呼语风格差异明显（文本相似度 < 0.6）
4. 前20字均为钩子类型（不含"您好"等模板句）
5. 在Chrome和Safari移动端均正常运行

### MVP验证指标（Day 7-10）

找20个真实求职者试用一周：
- **核心指标**：AI招呼语 vs 手写招呼语的回复率对比
- **达标线**：回复率提升 ≥ 15%
- **决策点**：达标 → 继续开发 | 不达标 → 果断止损

## 四、Prompt体系（核心资产）

### 招呼语生成 Prompt（Haiku，¥0.009/次）

```
## 角色
你是一位精通BOSS直聘生态的招聘沟通策略师。

## 核心认知
BOSS直聘HR的消息列表只显示前20字作为预览，这20字决定生死。
HR平均花2-3秒决定是否点开一条招呼语。

## 输入
- 用户简历摘要：{{resume_summary}}
- 目标岗位JD：{{jd}}
- 用户差异化特点：{{personal_highlight}}
- 风格偏好：{{style}}

## 钩子策略（6种，每个版本用一种）
1. 数字钩子："3年运营经验，从0做到月GMV 500万——"
2. 匹配钩子："看到贵司在招XX方向，我的经历高度匹配——"
3. 成就钩子："曾帮前司将YY指标提升300%——"
4. 问题钩子："贵司ZZ问题，我在上家刚好解决过——"
5. 关联钩子："一直在关注贵司新产品，发现我们在——"
6. 反模板钩子："我知道您每天收到100+条'您好'，所以——"

## 输出
生成3个版本的招呼语，每个版本：
- 钩子类型不同
- 总字数80-150字
- 前20字必须包含钩子元素
- 遵循"钩子句→价值主张→行动引导"三明治结构

## 禁止
- 禁止"您好/打扰了/冒昧"等套话
- 禁止空洞自我评价（"我学习能力强"——HR不信）
- 禁止超过200字
- 禁止没有具体数据或实例的泛泛表达
```

### Prompt质量评估维度

| 维度 | 权重 | 检查方法 |
|------|------|---------|
| 指令遵循度 | 35% | 字数、格式、禁止项扫描 |
| 内容质量 | 30% | 钩子类型判定、量化数据匹配 |
| 差异性 | 20% | 版本间余弦相似度 < 0.6 |
| 可用率 | 15% | 用户一键复制比例（线上数据） |

## 五、技术架构（修正版）

### 不做Chrome扩展的DOM操控，改用PWA方案

```
┌──────────────────────────────────────┐
│          PWA (移动端+桌面端)          │
│                                      │
│  ┌──────────────────────────────┐   │
│  │  输入层                       │   │
│  │  - JD文本粘贴                 │   │
│  │  - JD链接解析（可选）          │   │
│  │  - 用户背景管理               │   │
│  └──────────┬───────────────────┘   │
│             │                        │
│  ┌──────────▼───────────────────┐   │
│  │  AI调用层（客户端fetch）       │   │
│  │  - DeepSeek API（推荐默认）    │   │
│  │  - OpenAI / Claude（可选）     │   │
│  │  - 流式输出+Port通信          │   │
│  └──────────┬───────────────────┘   │
│             │                        │
│  ┌──────────▼───────────────────┐   │
│  │  输出层                       │   │
│  │  - 3版本招呼语展示            │   │
│  │  - 一键复制到剪贴板           │   │
│  │  - 历史记录（localStorage）   │   │
│  └──────────────────────────────┘   │
│                                      │
│  数据存储：localStorage（配置+历史）  │
│  零服务器：不需要后端                 │
│  零权限：不需要安装、不需要授权        │
└──────────────────────────────────────┘
```

### 为什么不用Chrome扩展（现阶段）

| 问题 | 说明 |
|------|------|
| **TAM太小** | BOSS用户80-95%在移动端，Web端天花板≈几千用户 |
| **CSP限制** | Content Script无法直接fetch AI API |
| **SW回收** | Service Worker 30s回收导致AI流式响应丢失 |
| **DOM竞态** | SPA页面切换导致注入UI时机不可靠 |
| **封号风险** | 任何DOM操控在平台检测中与全自动脚本同类 |
| **Chrome审核** | 需要$5注册费+1-5天审核，不适合快速验证 |

### 技术选型（MVP）

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | 纯HTML+CSS+JS（单文件） | 零依赖，手机直接打开 |
| AI API | DeepSeek V3（默认） | 便宜(¥0.5/百万token)，中文好，注册送500万token |
| 存储 | localStorage | 够用，无需数据库 |
| 部署 | GitHub Pages | 免费，HTTPS，PWA可安装 |
| 流式输出 | SSE（Server-Sent Events） | 原生支持，手机浏览器兼容 |

### 成本估算

| 项目 | 费用 |
|------|------|
| DeepSeek API（招呼语，Haiku级别替代） | ¥0.07/次 |
| 每天20条招呼语 | ¥1.4/天 |
| 月成本 | ≈¥42/月 |
| GitHub Pages托管 | ¥0 |

## 七、对你（罗南杨）的最终建议

### 优先级排序

```
第1优先级：投简历（每天上午，至少15份）
    ↓
第2优先级：准备面试（收到回复后）
    ↓
第3优先级：开发MVP（晚上/周末，2-3天出最简版）
```

### 关键提醒

1. **被"做工具"麻痹的陷阱**：做工具是舒适区（有掌控感），投简历是焦虑区（被拒绝）。警惕用"我正在做一个能帮我求职的工具"来拖延真正该做的事——**投出下一份简历**。

2. **2-3天最简版就够**：不需要Chrome扩展、不需要UI、不需要简历优化。浏览器控制台粘贴一段JS，输入JD链接→AI生成招呼语。这2-3天的投入产出比最高。

3. **如果时间实在紧张**：直接用ChatGPT/Claude对话窗口生成招呼语——每次复制JD进去，说"帮我写一段BOSS直聘的打招呼语"。这也是AI辅助求职，只是没有产品化。

4. **生存窗口**：存款不足4000元，深圳生存窗口约1.3-1.6个月。优先保证每天投15+份简历，工具只是辅助。
