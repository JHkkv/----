## 本周必知

### 1. Anthropic 在 B2B 市场首次反超 OpenAI

Ramp 的支出数据不会说谎：美国企业客户中 Anthropic 占比 34.4%，OpenAI 跌至 32.3%。一年前这个数字是 8% vs 45%。

这不是偶然。Anthropic 的产品矩阵正在形成"开发者 → 中小企业 → 大客户"的三级漏斗：Claude Code 占领开发者心智，刚刚推出的 Claude for Small Business 切入中小企业（集成 QuickBooks/PayPal/HubSpot，15 个开箱即用工作流），加上企业级 API。

**为什么值得关注**：这说明 AI 市场的竞争已从"谁的模型更强"转向"谁的产品更好用"。对普通人来说，意味着你的 AI 工具选择会越来越多，成本会越来越低。

### 2. Kimi K2：460 万美元训练出击败 GPT-5.5 的模型

Moonshot AI 杨植麟放出一个 40 分钟的视频，完整拆解 Kimi K2 的训练过程。核心突破是线性注意力架构创新——用聪明设计而非堆算力抹平资源差距。

**为什么值得关注**：这不是一个大厂的故事。这是一个小团队用巧思逆袭的故事。对于正在用 AI 做产品的人来说，它证明了一件事：资源不是决定性因素，创造力才是。

### 3. Claude 周限额提升 50% + 月度编程额度

两条 Claude 更新放在一起看才有意思。周限额提升 50%（持续到 7 月 13 日）是给个人用户的诚意，而月度编程额度（6 月 15 日起覆盖 Agent SDK + GitHub Actions）是给开发者的基建。

**为什么值得关注**：这意味着你可以用 Claude Code 跑 CI/CD 流水线、自动审代码、批量处理任务——而且不烧自己的用量。AI 编程工具正在从"手艺人工具"变成"工厂流水线"。

## 值得安利

### 这周发现的好东西

**OpenSquilla — 省 90% Token 的开源项目**

如果你在跑 AI 应用，Token 费用大概率是你最大的痛。OpenSquilla 的做法很聪明：自动判断任务复杂度，简单问题路由到便宜模型，复杂任务分给强模型。路由决策在本地完成，不消耗 Token。实际传输 Token 减少了 90% 以上。

👉 https://x.com/vista8/status/2054757474100760626

**psql_bm25s — PostgreSQL 检索提速 23 倍**

如果你的项目用 PostgreSQL 做知识库或搜索引擎，这个原生 BM25 访问方法能让检索不再成为瓶颈。开源、直接装扩展就能用。

👉 https://x.com/EMostaque/status/2054587062033043799

**微信群聊总结 Skill（Claude Code 用户专属）**

baoyu-skills 新增了一个微信相关的 Skill，基于 wx-cli 读取群聊数据，配合 Claude Code + Opus 4.6 效果最佳。适合需要跟进几十个群但没时间每条都看的运营人员。

👉 https://github.com/JimLiu/baoyu-skills/tree/main/skills/baoyu-wechat-summary

**Cursor 云端智能体环境**

支持多仓库环境、Dockerfile 配置，缓存命中后构建速度提升 70%。如果你在管理多个项目，Agent 协作的门槛大幅降低。

👉 https://cursor.com/blog/cloud-agent-development-environments

## 有趣角落

*日报里不会放、但值得一乐的事情*

## 一个思考

这周的主题是"AI 从技术变成基础设施"。

Anthropic 的王牌不是模型参数，而是产品矩阵。Kimi K2 证明的不是中国 AI 追上美国，而是小团队可以挑战巨头。Claude 的定价策略不是在打价格战，而是在建生态。
