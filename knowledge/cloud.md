# Cloud.md — AI 处理协议

## 一、核心原则

### 1.1 增量处理
- 每次仅处理 `raw/` 中**新增或更新**的材料（对比 `.processing-log.md`）
- 不重复处理已标记为 `done` 的文件
- 处理后必须在 `.processing-log.md` 中记录文件哈希和处理时间

### 1.2 概念提取
- 从每份材料中提取 3-8 个核心概念
- 每个概念需包含：定义、来源引用、关联已有概念
- 概念名称使用**名词短语**，简洁可检索

### 1.3 双向链接
- 所有概念条目使用 Obsidian 兼容的 `[[wiki/概念名]]` 链接
- 新增概念时，必须检查是否与已有概念关联，并在已有概念的页面中**回链**
- 关联类型标注：`#延伸` `#对比` `#前提` `#应用`

### 1.4 溯源引用
- 每个提取的知识点标注来源：`> 来源：raw/文件名`
- 关键引用保留原文（blockquote 格式）

---

## 二、处理流程

```
raw/新文件 → AI 读取 → 提取核心概念 
    → 对比已有 Wiki 索引 → 创建/更新概念页（含双向链接）
    → 更新 Wiki 索引 → 记录处理日志 → 完成
```

## 三、文件格式规范

### 3.1 Raw 材料命名
```
{日期}-{来源类型}-{简短标题}.md
例：2026-05-12-文章-Karpathy知识复利.md
```

### 3.2 Wiki 概念页模板
```markdown
---
aliases: [别名1, 别名2]
tags: [标签1, 标签2]
source: "[[../raw/源文件名]]"
created: YYYY-MM-DD
---

# 概念名

## 定义
一句话定义 + 详细说明

## 来源
> 原文摘录

## 关联概念
- [[概念A]] #延伸
- [[概念B]] #前提

## 我的思考
（用户可补充）
```

### 3.3 输出内容模板
```markdown
---
type: output
based_on: [[wiki/概念A]], [[wiki/概念B]]
created: YYYY-MM-DD
---

# 输出标题

## 核心观点
...

## 论证/展开
...

## 引用来源
- [[../raw/源文件1]]
- [[../raw/源文件2]]
```

## 四、特殊规则

### 4.1 闭环回存
当 AI 生成的回答质量高、具有复用价值时，用户可命令"回存"，将回答存入 `raw/` 作为新输入材料，标记 `type: ai-generated-feedback`

### 4.2 冲突处理
如果新概念与已有概念矛盾：
- 不覆盖已有概念
- 在新概念页中创建 `## 观点对比` 段落
- 在已有概念页中添加 `#对比` 回链

### 4.3 定期整理
每周或积累 20+ 概念后，提示用户是否进行：
- 合并相似概念
- 清理过时信息
- 重构索引分类

---

## 五、AI 行为指令

当用户说"处理知识"或"处理 [文件名]"时：
1. 读取 cloud.md 本文件
2. 读取 `.processing-log.md` 确认待处理文件
3. 逐个处理新文件，严格按本协议执行
4. 完成后报告：处理了 X 个文件，提取了 Y 个新概念，关联了 Z 个已有概念
