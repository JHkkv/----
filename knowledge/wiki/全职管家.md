# 全职管家

## 定义
大模型在个人知识库中的理想角色——全天候自动整理、关联、管理知识，如同全职管家。

## 来源
> 让大模型成为全职管家，自动整理关联新存入信息，打造能自动成长的知识复利第二大脑。

## 应具备的能力
1. 自动阅读并理解新存入的材料
2. 提取核心概念并结构化
3. 与已有知识建立关联
4. 维护知识索引
5. 基于知识库回答溯源问题
6. 将高质量输出回存为新的知识输入

## 现实约束
- Claude Code 不是常驻进程，需要用户主动触发
- 但 memory 系统和知识库结构可弥补非持续性
- 每次对话都能继承之前积累的 Wiki 索引
